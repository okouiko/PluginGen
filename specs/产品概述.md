# 产品概述 — PluginGen

## 1. 项目愿景 (Vision)

> 为 Minecraft 插件的需求者与创造者之间架起一座桥梁——让任何有创意的人，无需配置 Java/Maven/IDE，在浏览器中就能完成从「我有一个想法」到「一个可运行的插件」的全过程。

## 2. 核心价值 (Value Proposition)

- **用户价值**：消除 Minecraft 插件开发的环境配置门槛，将 AI 生成、在线编译、版本管理、社区分享整合为单一闭环体验。服主和玩家从「想做一个功能」到「拿到一个 .jar」的时间从数天缩短到数分钟。
- **技术/商业价值**：构建「AI 生成 → 在线编辑 → 沙箱编译 → 社区发布」的全流程自动化链路，形成插件生产的 UGC 平台。创作者体系与作品广场形成社区飞轮效应。

## 3. 核心板块 (Core Modules)

### 3.1 AI 插件引擎

- **描述**：接收用户需求输入（插件名称、MC 版本、服务端核心、功能描述等），调用 deepseek-v4-flash 生成完整可编译的 Maven 项目结构。覆盖 Bukkit/Spigot/Paper/Purpur/BungeeCord/Velocity 六大核心，自动匹配对应 API 与 Java 版本。
- **关键特性**：
  - 多核心项目骨架生成（主类、命令类、事件监听器、管理器、plugin.yml/config.yml/pom.xml）
  - AI 代码逐文件在线解释（面向新手的教学辅助）
  - WebSocket 实时推送生成与编译进度

### 3.2 在线开发工作台

- **描述**：在浏览器中完成插件的二次编辑、版本管理和编译检测，无需任何本地工具。
- **关键特性**：
  - 在线源码查看与编辑器（语法高亮、文件树浏览）
  - 增量修改能力（追加功能、修改逻辑），无需重新全量生成
  - 历史版本管理：每次编辑保存快照，支持回退和差异对比
  - Docker 沙箱编译检测：自动发现语法错误、缺失 import、类型不匹配、API 调用错误、plugin.yml 路径错误
  - 自动修复 + 重新编译 + 错误详情展示

### 3.3 用户系统与创作者生态

- **描述**：完整的用户账户体系与创作者成长机制，支撑平台运营与社区活跃度。
- **关键特性**：
  - 邮箱 + 密码注册登录（JWT 会话管理）
  - 个人主页：资料、简介、发布清单、置顶作品、数据统计（下载量/收藏量/获赞数）
  - 创作者等级与经验值系统（完成生成、发布、获得点赞等行为积累经验）
  - 每日 20 次 AI 生成配额
  - 每日签到与任务系统（如：今日生成 1 个插件、发布 1 个作品）
  - 生成历史管理、插件项目管理

### 3.4 作品广场与社区

- **描述**：插件的公开市场与社交空间，让插件从「个人工具」变成「社区资产」。
- **关键特性**：
  - 插件公开浏览：列表/详情页，展示介绍、作者、热度（下载/收藏/点赞）、评分
  - 搜索与筛选（按核心类型、MC 版本、标签等）
  - 收藏、点赞、评论、评分（1-5 星）
  - 关注创作者
  - 站内私信系统
  - 通知系统（被点赞、被评论、被关注、新粉丝等）

### 3.5 下载与依赖管理

- **描述**：提供源码与编译产物的灵活分发，以及外部依赖的上传支持。
- **关键特性**：
  - 源码 ZIP 下载
  - 编译 JAR 下载
  - 上传外部依赖 JAR（经济插件 API、权限插件 API 等）辅助编译
  - 上传已有源码 ZIP 自动分析和修复编译问题

## 4. 目标用户 & 核心场景 (User & Scenes)

### 4.1 目标用户

- **服务器服主**：运营着自己的 Minecraft 服务器，需要定制化插件（如签到系统、商店系统）但不会 Java 编程。需要一个工具将「功能需求」直接转化为「可用插件」。
- **插件开发新手**：对 Minecraft 插件开发有兴趣但被 Java 环境配置、Maven 构建、IDE 搭建阻碍。需要一个低门槛的入门环境，配合 AI 解释学习代码。
- **普通玩家**：有创意想法但无技术背景，想把自己在服务器里的构想变成实际插件并分享给社区。

### 4.2 核心场景 (Key Use Cases)

- **场景 1（服主快速定制）**：服主登录 PluginGen，填写「每日签到插件，Minecraft 1.20.1，Paper 核心」，描述功能需求。AI 生成完整项目，经在线编译检测通过后直接下载 JAR，上传到服务器使用。
- **场景 2（新手学习与迭代）**：新手用户查看 AI 生成的代码，通过 AI 解释功能逐行理解命令注册和事件监听。尝试追加一个「连续签到奖励」功能，系统增量修改并生成新版本，前后对比查看 diff。
- **场景 3（社区分享）**：开发者完成插件后发布到作品广场，其他用户搜索发现、下载源码学习、点赞评论交流。创作者通过积累经验和作品获得等级提升。
- **场景 4（遗留代码修复）**：用户上传自己本地写的残缺源码 ZIP，系统分析并修复编译问题，提供修复后的可下载 JAR。

## 5. 关键业务流程 (Core Workflows)

### 流程 1：AI 插件生成主链路

```
用户填写表单 → 调用 DeepSeek API 生成代码 →
  → WebSocket 推送「生成中…」进度 →
  → 保存项目到数据库 + 写入本地磁盘 →
  → Docker 容器内执行 Maven compile →
  → 检测编译结果（成功/失败 + 错误信息） →
  → WebSocket 推送结果 → 用户查看/下载
```

### 流程 2：增量修改与版本管理

```
用户查看当前代码 → 提交修改描述 →
  → AI 基于上下文生成 diff 代码 →
  → 保存为新版本快照 →
  → 触发 Docker 编译检测 →
  → 用户可回退到任意历史版本 / 对比版本差异
```

### 流程 3：社区交互链路

```
用户公开发布插件 → 进入作品广场 →
  → 其他用户浏览/搜索/筛选 →
  → 收藏/点赞/评论/评分 →
  → 被交互用户收到通知 →
  → 创作者获得经验值 → 等级成长
```

## 6. 验收标准 (Acceptance Criteria)

### 功能完备性

- ✅ AI 可成功生成 Bukkit/Spigot/Paper 至少 3 种核心的可编译项目，生成结构包含主类、命令、事件监听、plugin.yml、config.yml、pom.xml
- ✅ 在线源码查看、AI 解释功能可用
- ✅ 增量修改可正确追加代码，历史版本可回退
- ✅ Docker 沙箱编译流程走通，编译成功生成 JAR，编译失败展示错误详情
- ✅ 源码 ZIP 和 JAR 下载正常
- ✅ 邮箱注册/登录、JWT 鉴权、每日配额计数正常
- ✅ 作品广场：发布、浏览、搜索、筛选、收藏、点赞、评论、评分正常
- ✅ 个人主页：信息展示、作品列表、数据统计正常
- ✅ 签到与每日任务功能可用
- ✅ BungeeCord/Velocity 核心至少一种额外核心可生成

### 技术指标

- ✅ 页面首次加载 ≤ 2s（未计 AI 调用时间）
- ✅ AI 生成阶段时长 ≤ 30s（通过 WebSocket 实时反馈进度）
- ✅ Docker 编译时长 ≤ 60s
- ✅ PostgreSQL 查询响应 ≤ 200ms（99% 分位）
- ✅ 支持至少 50 并发用户

## 7. 约束条件 (Constraints)

- **技术栈**：前端 React + TailwindCSS + Shadcn/ui，后端 Node.js，数据库 PostgreSQL
- **AI 模型**：deepseek-v4-flash，需通过 API 调用（非本地部署）
- **编译环境**：Docker 沙箱，每请求独立容器，容器内预装 Maven + 对应 JDK（8/11/16/17/21）
- **存储**：本地磁盘存储生成文件（源码、JAR、ZIP）
- **部署**：Linux 服务器全托管（自备服务器）
- **UI 规范**：严格遵循 DESIGN.md 定义的 Anthropic Claude 风格配色、排版、组件与布局体系
- **界面语言**：仅支持中文
- **收费模式**：暂不收费，每日免费 20 次生成配额
- **文件安全**：上传的 JAR/ZIP 不做主动安全检查（编译在 Docker 沙箱中隔离执行）

## 8. 项目边界 (Out of Scope)

本阶段不涉及以下内容：

- ❌ 多语言国际化支持（暂仅中文）
- ❌ 移动端 App（仅 Web）
- ❌ 第三方 OAuth 登录（仅邮箱+密码）
- ❌ 付费套餐订阅系统
- ❌ 自动审核机制（插件直接发布）
- ❌ 第三方存储服务（如 OSS/S3，暂用本地磁盘）
- ❌ Mod 开发支持（Forge/NeoForge/Fabric，仅限插件）
- ❌ 插件安装到服务器的自动部署功能
- ❌ 图床/富文本编辑器（评论支持纯文本）
- ❌ 数据导出/分析仪表盘
