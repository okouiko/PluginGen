# 技术栈选型 — PluginGen

## 1. 核心决策 (Key Decisions)

**为什么选择这套技术栈？**

- **NestJS（而非 Express）**：PluginGen 包含 AI 生成、编译管理、社区交互、用户体系、通知系统 5 大业务模块，NestJS 的模块化架构（Module/Controller/Service/Guard/Interceptor）天然适合这种中大型项目的分层组织。内置的 WebSocket Gateway 和依赖注入系统减少了 40% 以上的胶水代码。Express 对于此规模的项目过于"裸"，需要开发者自己拼装架构，增加维护成本。
- **Prisma（而非 TypeORM）**：全类型安全、Schema-first 建模、自动 Migration 管理，与 NestJS 的 TypeScript 生态同为"约定优于配置"的设计哲学，配合度最高。
- **Redis 作为中间件**：三大用途——① WebSocket 连接路由 ② Bull 任务队列后端（编译作业调度）③ 每日配额计数器。一份 Redis 实例解决三个关键需求，性价比极高。
- **Monaco Editor**：直接集成 VS Code 的编辑器内核，零成本获得语法高亮、代码折叠、多光标编辑、DiffEditor（版本对比）。对比 CodeMirror 功能更全面，且编辑器交互是本平台的核心体验。
- **Dockerode + 预构建镜像**：而非运行时安装 Maven/JDK。每个 JDK 版本（8/11/16/17/21）预编译为 Docker 镜像，编译容器启动时间从分钟级降至秒级。

## 3. 后端 (Backend)

| 层次               | 选择                                    | 版本     | 选型理由                                                                                                                                                 |
| ------------------ | --------------------------------------- | -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **运行时**         | Node.js + TypeScript                    | 22 LTS   | 用户已选定。全程类型安全，I/O 密集型任务适合非阻塞模型                                                                                                   |
| **Web 框架**       | NestJS                                  | 11.x     | 模块化架构（Auth/Plugin/Compile/Community/Notification 各为独立 Module），内置 WebSocket Gateway、依赖注入、Guards/Pipes/Interceptors 企业级模式         |
| **API 风格**       | RESTful                                 | —        | 社区功能以 CRUD 为主，REST 足够。不增加 GraphQL 的复杂度                                                                                                 |
| **ORM / 数据库层** | Prisma                                  | 6.x      | Schema-first 建模，全类型安全客户端，自动 Migration 管理，最佳 PostgreSQL + Node.js 组合                                                                 |
| **WebSocket**      | `@nestjs/websockets` + `ws`             | built-in | NestJS 原生集成，支持 Gateway/Room/Event 模式，天然线程安全                                                                                              |
| **认证框架**       | `@nestjs/passport` + `passport-jwt`     | latest   | NestJS 原生 Passport 集成，local + JWT 双策略                                                                                                            |
| **密码哈希**       | bcrypt                                  | latest   | 业界标准，对低频率注册场景完全足够                                                                                                                       |
| **数据校验**       | `class-validator` + `class-transformer` | latest   | NestJS DTO 原生校验，与 `@nestjs/swagger` 配合自动生成 API 文档                                                                                          |
| **任务队列**       | Bull (`@nestjs/bull`)                   | latest   | **关键组件**。Docker 编译是异步耗时操作（平均 20-60s），Bull 基于 Redis 提供可靠的作业调度、重试、延迟和进度追踪。WebSocket 通过作业事件实时推送编译进度 |
| **AI 调用**        | OpenAI SDK (`openai`)                   | latest   | DeepSeek API 完全兼容 OpenAI 格式，直接复用 SDK                                                                                                          |
| **Docker 管理**    | Dockerode                               | latest   | Node.js 原生 Docker 远程 API SDK。编译流程：`createContainer` → `attach` → `start` → `wait` → 获取日志                                                   |

| **Diff 引擎**      | `diff` (jsdiff)                         | latest   | 生成统一格式代码差异                                                                                                                                     |
| **邮件服务**       | Nodemailer                              | latest   | 邮箱验证/密码重置（预留）                                                                                                                                |
| **API 文档**       | `@nestjs/swagger` (Swagger UI)          | latest   | 自动生成 OpenAPI 文档，前后端联调必备                                                                                                                    |
