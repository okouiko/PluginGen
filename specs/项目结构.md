# 项目结构 — PluginGen

## 1. 架构概览

采用 **Monorepo** 布局（pnpm workspace），前后端分离但共享一个仓库。根目录承载基础设施配置文件，`frontend/` 和 `backend/` 各自独立编译与部署。

```
PluginGen/
├── frontend/          # Vite + React SPA（已构建好的 dist/ 由 Nginx 直接托管）
├── backend/           # NestJS API 服务（含 WebSocket Gateway、Bull 队列）
├── docker/            # Docker 编译沙箱镜像定义
├── nginx/             # Nginx 反向代理和静态资源配置
├── scripts/           # 部署/构建脚本
├── specs/             # 项目定义文档（产品概述、技术栈、项目结构等）
└── .github/           # GitHub Actions CI/CD
```

## 3. 模块详细说明

### 3.1 `frontend/src/pages/` — 页面路由

```
路由路径                           → 页面组件
/                                  → Home.tsx（首页）
/login                             → Login.tsx
/register                          → Register.tsx
/dashboard                         → DashboardLayout.tsx（需登录）
/dashboard/create                  → CreatePlugin.tsx（AI 生成表单）
/dashboard/plugins                 → PluginList.tsx（我的插件列表）
/dashboard/plugins/:id/edit        → PluginEdit.tsx（编辑器）
/dashboard/settings                → Settings.tsx
/workspace                         → WorkspacePage.tsx（作品广场）
/workspace/:id                     → PluginDetail.tsx（插件详情）
/user/:id                          → Profile.tsx（个人主页）
/messages                          → MessagesPage.tsx（站内信）
/daily                             → DailyPage.tsx（签到/任务）
```

### 3.2 `backend/src/modules/` — 9 大业务模块

| 模块             | 职责                                     | 关键 API                                                                   |
| ---------------- | ---------------------------------------- | -------------------------------------------------------------------------- |
| **auth**         | 注册、登录、JWT 签发                     | `POST /auth/register`, `POST /auth/login`                                  |
| **user**         | 用户资料、等级、经验值、每日配额         | `GET /user/profile`, `PATCH /user/profile`                                 |
| **plugin**       | 项目 CRUD、版本管理、发布                | `POST /plugins`, `GET /plugins/:id/versions`, `PATCH /plugins/:id/publish` |
| **ai-generator** | DeepSeek API 调用、Prompt 工程、代码解释 | `POST /ai/generate`, `POST /ai/modify`, `POST /ai/explain`                 |
| **compile**      | Docker 编译调度、自动修复                | `POST /compile/start` → Bull Queue → WebSocket 推送进度                    |
| **community**    | 作品广场、收藏、评论、评分、关注         | `GET /community/plugins`, `POST /community/favorite`                       |
| **notification** | WebSocket 通知、站内私信                 | WebSocket 实时推送 + `GET /notifications`                                  |
| **daily**        | 签到、每日任务                           | `POST /daily/checkin`, `GET /daily/tasks`                                  |
| **file**         | 文件上传、ZIP 打包下载                   | `POST /file/upload`, `GET /file/download/:id`                              |

### 3.3 `backend/src/common/` — 共享基础设施

| 层级             | 职责          | 关键文件                                                              |
| ---------------- | ------------- | --------------------------------------------------------------------- |
| **guards**       | 路由权限控制  | `jwt-auth.guard.ts`（注入 JWT 用户），`throttle.guard.ts`（配额检查） |
| **decorators**   | 参数装饰器    | `@CurrentUser()` 获取当前登录用户                                     |
| **filters**      | 全局异常处理  | `all-exceptions.filter.ts`（统一错误响应格式）                        |
| **interceptors** | 请求/响应拦截 | `transform.interceptor.ts`（统一 `{ code, data, message }` 响应）     |
| **pipes**        | 参数校验      | `validation.pipe.ts`（基于 class-validator）                          |
| **prisma**       | 数据库连接    | `prisma.service.ts`（NestJS 单例，全局复用）                          |

### 3.4 `docker/compiler/` — 编译沙箱镜像

每个 JDK 版本对应一个 Dockerfile，镜像预装：

- 对应版本 OpenJDK
- Maven 3.9+
- 常用 Maven 仓库缓存（Spigot/Bukkit/PaperAPI 等）
- 构建脚本入口（接收源码路径 → `mvn clean package` → 输出 JAR 路径）

## 4. 文件放置规则

| 文件类型        | 放置位置                                                      | 禁止违反                                             |
| --------------- | ------------------------------------------------------------- | ---------------------------------------------------- |
| **API 路由**    | `modules/[module]/controller.ts`                              | 禁止直接在 `main.ts` 或 `app.module.ts` 中写路由逻辑 |
| **业务逻辑**    | `modules/[module]/service.ts`                                 | 禁止写在 Controller 或 Gateway 中                    |
| **数据库操作**  | 通过 `prisma.service.ts` 在 Service 层完成                    | 禁止直接在其他层调用 Prisma Client                   |
| **DTO 校验**    | `modules/[module]/dto/*.dto.ts`                               | 禁止在 Controller 参数中直接写 inline 类型           |
| **页面组件**    | `pages/[page].tsx` 或 `pages/[area]/[page].tsx`               | 禁止在组件中写业务逻辑（应抽取到 hooks 或 stores）   |
| **可复用 UI**   | `components/ui/*.tsx`（Shadcn/ui）、`components/shared/*.tsx` | 禁止在业务页面中重复编写相同的 UI 逻辑               |
| **前端类型**    | `types/*.ts` 对应领域                                         | 禁止随处定义 `interface`（应集中在 types 目录）      |
| **环境变量**    | `backend/.env`（本地）、`backend/.env.example`（模板）        | 禁止提交 `.env` 到 Git 仓库                          |
| **测试文件**    | 与源文件同目录：`xxx.spec.ts` 或 `xxx.test.ts`                | 禁止将测试文件散落在其他目录                         |
| **Prompt 模板** | `modules/ai-generator/prompts/`                               | 禁止将 Prompt 字符串硬编码在 Service 中              |
