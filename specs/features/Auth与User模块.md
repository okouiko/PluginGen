# Auth & User 模块 — 功能需求说明书

## 1. 背景与价值

**为什么做**：Phase 1.1 + 1.2 已搭好项目骨架和公共基础设施，但用户还不能注册登录。Auth 模块是平台的入口——没有它，用户无法创建插件、发布作品或进行任何社交互动。User 模块则承载了用户的身份标识和成长体系。

**做成什么样**：用户打开网站 → 在首页点击登录/注册 → 填写邮箱、昵称和密码 → 注册成功后自动进入工作台 → 可以查看和编辑自己的个人资料 → 在导航栏看到自己的昵称和每日剩余次数。

**关联**：路线图 Phase 1.3 / 里程碑一（MVP）

## 2. 范围

### 本次做

- 后端：邮箱 + 密码注册（`POST /auth/register`）
- 后端：邮箱 + 密码登录（`POST /auth/login`）→ 签发 JWT
- 后端：Passport JWT 策略（从 `Authorization: Bearer` 解析 Token，注入 `req.user`）
- 后端：获取当前用户信息（`GET /user/profile`）
- 后端：修改个人资料（`PATCH /user/profile`）— 昵称、个人简介
- 后端：创作者等级与经验值计算（`user-level.service.ts`）
- 后端：每日生成配额中间件（Redis TTL）
- 前端：登录页（邮箱 + 密码 + 登录按钮 + 跳转注册链接）
- 前端：注册页（邮箱 + 昵称 + 密码 + 确认密码 + 注册按钮）
- 前端：注册成功后自动登录并跳转到工作台
- 前端：登录成功后跳转到工作台
- 前端：顶部导航栏显示用户昵称、等级徽章、每日剩余次数
- 前端：导航栏下拉菜单（个人主页 / 设置 / 登出）
- 前端：登出 → 清除 Token → 跳转到首页
- 前端：未登录访问受保护页面 → 自动跳转到登录页
- 前端：`hooks/use-auth.ts` — 封装登录、注册、登出、获取当前用户

### 本次不做

- 邮箱验证（注册后直接可用，不发送验证邮件）— 后续迭代
- 密码重置（忘记密码功能）— 后续迭代
- 第三方 OAuth 登录（GitHub/微信等）— 已确认不在此阶段
- 用户头像上传 — 已确认 MVP 不包含
- 个人主页公开页面（展示作品列表等）— Phase 2.3 再做
- 管理员后台 — 项目边界已排除

## 4. 需求详述

### 4.1 核心流程

#### 流程 1：注册

1. 新用户点击首页或导航栏的"注册"按钮 → 进入 `/register` 页面
2. 用户填写**邮箱**、**昵称**、**密码**、**确认密码**
3. 前端立即校验：邮箱格式正确、昵称 2-20 字符、密码 ≥ 8 位且含字母+数字、两次密码一致
4. 提交后系统检查邮箱是否已被注册
5. 通过后创建用户（密码 bcrypt 哈希存储），签发 JWT Token
6. 自动登录：前端保存 Token，跳转到工作台 `/dashboard`
7. 导航栏更新显示用户昵称和默认头像

#### 流程 2：登录

1. 用户在登录页填写**邮箱**和**密码**
2. 提交后系统验证邮箱存在 + 密码匹配
3. 匹配成功 → 签发 JWT Token → 前端保存 → 跳转 `/dashboard`
4. 匹配失败 → 显示"邮箱或密码错误"提示

#### 流程 3：登出

1. 用户点击导航栏下拉菜单中的"登出"
2. 前端清除 JWT Token 和用户信息
3. 跳转到首页

#### 流程 4：查看/编辑个人资料

1. 用户点击导航栏下拉菜单 → "个人主页"或"设置"
2. 显示当前资料：邮箱（只读）、昵称（可编辑）、个人简介（可编辑）、等级、经验值、每日剩余次数
3. 用户修改昵称或简介 → 提交 → 后端保存 → 前端更新显示

### 4.2 交互规则

- 注册成功后不展示"注册成功"弹窗，无缝切入已登录状态并跳转工作台
- 登录失败时，输入框不清空，仅显示错误提示："邮箱或密码错误"
- 登出后清除全部本地状态（Token + 用户信息），跳转到首页
- 未登录用户访问 `/dashboard`、`/daily`、`/messages` 等受保护页面时，自动跳转到 `/login`，并在登录后回跳原始页面
- 导航栏用户昵称旁显示等级徽章（如 Lv.3）和每日剩余次数（如 "剩余 15/20"）
- 密码输入框需支持显示/隐藏切换（眼睛图标按钮）

### 4.3 业务规则

| 编号   | 规则                                                                                                  |
| ------ | ----------------------------------------------------------------------------------------------------- |
| BR-001 | 邮箱：有效格式（`*@*.*`），唯一不可重复注册，不区分大小写（存储时统一转小写）                         |
| BR-002 | 昵称：2-20 个字符，不允许纯数字、纯特殊符号，不允许包含敏感词                                         |
| BR-003 | 密码：最少 8 位，至少 1 个字母 + 1 个数字，数据库中存储 bcrypt 哈希（不可逆）                         |
| BR-004 | JWT Token 有效期 7 天，过期后需重新登录                                                               |
| BR-005 | 等级初始值 1，经验值初始值 0。经验值获取：生成插件 +10，发布作品 +20，获赞 +5，被收藏 +3，每日签到 +5 |
| BR-006 | 每日生成配额：20 次（Redis TTL 管理，每天 0 点重置）                                                  |
| BR-007 | 用户只能修改自己的资料，不能查看或修改其他用户的资料                                                  |

## 5. 验收标准

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001**: 用户成功注册
  - **Given** 用户访问 `/register` 页面并填写邮箱 `test@example.com`、昵称 `测试玩家`、密码 `Abc12345`、确认密码 `Abc12345`
  - **When** 用户点击"注册"按钮
  - **Then** 注册成功，自动登录且跳转到 `/dashboard`，导航栏显示昵称 `测试玩家` 和默认头像

- [ ] **AC-002**: 用户成功登录
  - **Given** 用户已有账号（邮箱 `test@example.com`，密码 `Abc12345`）且在 `/login` 页面
  - **When** 用户输入正确邮箱和密码并点击"登录"
  - **Then** 登录成功，跳转到 `/dashboard`，导航栏显示用户昵称

- [ ] **AC-003**: 注册时密码明文在前端不可见
  - **Given** 用户在注册页输入密码
  - **When** 输入密码
  - **Then** 密码显示为 `••••••••`，可点击眼睛图标切换显示/隐藏

- [ ] **AC-004**: 登录后 JWT 存在本地
  - **Given** 用户成功登录
  - **When** 检查 localStorage 或 Zustand store
  - **Then** 存在有效的 JWT Token（非空字符串），且请求 API 时自动注入 `Authorization` 头

- [ ] **AC-005**: 导航栏显示用户信息
  - **Given** 用户已登录
  - **When** 查看页面顶部导航栏
  - **Then** 显示用户昵称、等级徽章（如 `Lv.1`）、每日剩余次数（如 `剩余 20/20`）

- [ ] **AC-006**: 登录后访问受保护页面
  - **Given** 用户已登录
  - **When** 直接在浏览器地址栏输入 `/dashboard`
  - **Then** 正常显示工作台页面，不会跳转到登录页

- [ ] **AC-007**: 注册后数据正确存储
  - **Given** 用户成功注册
  - **When** 查询数据库 `User` 表
  - **Then** 新增一条记录，`email` 为 `test@example.com`（小写）、`nickname` 为 `测试玩家`、`password` 为 bcrypt 哈希值（非明文）、`level` 为 `1`、`exp` 为 `0`、`dailyQuota` 为 `20`

- [ ] **AC-008**: 用户修改个人资料
  - **Given** 用户已登录且在资料编辑页面
  - **When** 将昵称改为"新昵称"，个人简介改为"我的 Minecraft 插件之旅"，点击保存
  - **Then** 资料更新成功，导航栏昵称同步更新为"新昵称"

- [ ] **AC-009**: 导航栏下拉菜单
  - **Given** 用户已登录
  - **When** 点击导航栏用户昵称区域
  - **Then** 弹出下拉菜单，包含"个人主页"、"设置"和"登出"三个选项

- [ ] **AC-010**: 用户登出
  - **Given** 用户已登录
  - **When** 点击下拉菜单中的"登出"
  - **Then** Token 被清除，页面跳转到首页，导航栏恢复为"登录"和"注册"按钮

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-101**: 邮箱已被注册
  - **Given** 邮箱 `test@example.com` 已存在
  - **When** 使用同一邮箱再次注册
  - **Then** 注册失败，显示错误提示"该邮箱已被注册"，输入内容保留

- [ ] **AC-102**: 密码强度不足
  - **Given** 用户在注册页
  - **When** 输入密码 `12345678`（仅数字，无字母）
  - **Then** 前端立即显示提示"密码至少包含 1 个字母和 1 个数字"，注册按钮不可点击

- [ ] **AC-103**: 昵称过短或过长
  - **Given** 用户在注册页
  - **When** 输入昵称为空或超过 20 个字符
  - **Then** 前端立即显示提示"昵称 2-20 个字符"，注册按钮不可点击

- [ ] **AC-104**: 两次密码不一致
  - **Given** 用户在注册页
  - **When** 密码输入 `Abc12345`，确认密码输入 `Abc12346`
  - **Then** 前端立即显示提示"两次密码不一致"，注册按钮不可点击

- [ ] **AC-105**: 登录时邮箱不存在
  - **Given** 用户输入的邮箱未注册
  - **When** 点击登录
  - **Then** 登录失败，显示"邮箱或密码错误"（不区分是邮箱不存在还是密码错误，防止邮箱枚举攻击）

- [ ] **AC-106**: 登录时密码错误
  - **Given** 用户邮箱存在但密码错误
  - **When** 点击登录
  - **Then** 登录失败，显示"邮箱或密码错误"（与 AC-105 相同提示）

- [ ] **AC-107**: 超过每日配额
  - **Given** 用户已使用完当日 20 次生成配额
  - **When** 尝试提交 AI 生成表单
  - **Then** 显示提示"今日生成次数已用完，明天再来吧"，生成按钮置灰不可点击

- [ ] **AC-108**: 未登录访问受保护页面
  - **Given** 用户未登录
  - **When** 直接访问 `/dashboard`
  - **Then** 自动跳转到 `/login?redirect=/dashboard`，登录后回跳至原始页面

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-201**: 邮箱不区分大小写 ← BR-001
  - **Given** 已注册邮箱 `Test@Example.com`
  - **When** 使用 `test@example.com` 登录
  - **Then** 登录成功（注册时已转为小写，登录时也转为小写再比对）

- [ ] **AC-202**: 密码哈希存储 ← BR-003
  - **Given** 任意已注册用户
  - **When** 查询数据库 `User` 表的 `password` 字段
  - **Then** 值以 `$2b$` 开头（bcrypt 哈希前缀），长度 60 字符，无法直接反解为明文密码

- [ ] **AC-203**: Token 过期行为 ← BR-004
  - **Given** 用户持有已过期（超过 7 天）的 JWT Token
  - **When** 调用需要认证的 API
  - **Then** 返回 401 + "Token expired"，前端清除 Token 并跳转到 `/login`

- [ ] **AC-204**: 用户只能修改自己的资料 ← BR-007
  - **Given** 用户 A 已登录
  - **When** 尝试通过 API 修改用户 B 的资料
  - **Then** 返回 403 Forbidden

- [ ] **AC-205**: 经验值计算准确 ← BR-005
  - **Given** 新注册用户（level=1, exp=0）
  - **When** 生成 1 个插件（+10 exp）
  - **Then** 用户 `exp` 变为 `10`，`level` 仍为 `1`（未达升级阈值）
