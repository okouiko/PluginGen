# 作品广场 — 技术设计文档

## 1. 设计概要

**功能描述**：实现 PluginGen 的社区核心——作品广场，支持插件的发布/下架、搜索筛选、收藏/点赞/评论/评分、关注创作者等社交互动功能。

**影响范围**：后端新增 `community/` 模块；前端新增 `pages/workspace/WorkspacePage.tsx`、`pages/workspace/PluginDetail.tsx`、`components/community/*` 系列组件；修改 `pages/dashboard/PluginEdit.tsx` 增加发布入口。

**技术难点**：

- 搜索功能（全文搜索 + 多条件筛选）需要合理设计 Prisma 查询和索引
- 评分均分的实时计算和缓存策略

**外部依赖**：无新增（使用已有 Prisma 数据库操作）

```typescript
// rating.service.ts — upsertRating()
async upsertRating(userId: string, pluginId: string, score: number): Promise<RatingResult> {
  const plugin = await this.prisma.pluginProject.findUnique({ where: { id: pluginId } });

  // 阻止自评 → AC-102
  if (plugin.userId === userId) {
    throw new BadRequestException('Cannot rate your own plugin');
  }

  // Upsert: 同一用户同一插件只有一条评分记录
  const rating = await this.prisma.rating.upsert({
    where: { userId_pluginId: { userId, pluginId } },
    update: { score },
    create: { userId, pluginId, score },
  });

  // 重新计算均分
  const aggregate = await this.prisma.rating.aggregate({
    where: { pluginId },
    _avg: { score: true },
    _count: { score: true },
  });

  return {
    score: rating.score,
    avgRating: aggregate._avg.score || 0,
    ratingCount: aggregate._count.score,
  };
}
```

## 5. 核心逻辑

### 5.1 发布流程 → AC-001

```typescript
// community.service.ts — publish()
async publish(pluginId: string, userId: string, dto: PublishDto) {
  const plugin = await this.prisma.pluginProject.findUnique({ where: { id: pluginId } });

  // 检查所有权
  if (plugin.userId !== userId) throw new ForbiddenException();

  // 检查编译状态
  if (plugin.status !== 'COMPILED') {
    throw new BadRequestException('Plugin must be compiled before publishing');  // → AC-103
  }

  // 更新发布状态
  const updated = await this.prisma.pluginProject.update({
    where: { id: pluginId },
    data: {
      isPublished: true,
      description: dto.description,
    },
  });

  // 奖励经验值
  await this.userLevelService.addExp(userId, 20);  // 发布 +20 exp

  // 通知粉丝（异步）
  await this.notificationService.notifyFollowers(userId, 'plugin.published', {
    pluginId,
    pluginName: plugin.name,
  });

  return updated;
}
```

### 5.2 收藏切换 → AC-007

```typescript
// favorite.service.ts — toggle()
async toggle(userId: string, pluginId: string): Promise<{ favorited: boolean; favoriteCount: number }> {
  const existing = await this.prisma.favorite.findUnique({
    where: { userId_pluginId: { userId, pluginId } },
  });

  if (existing) {
    // 取消收藏
    await this.prisma.favorite.delete({ where: { id: existing.id } });
    await this.prisma.pluginProject.update({
      where: { id: pluginId },
      data: { favoriteCount: { decrement: 1 } },
    });
    return { favorited: false, favoriteCount: (await this.getCount(pluginId)).favoriteCount };
  } else {
    // 收藏
    await this.prisma.favorite.create({ data: { userId, pluginId } });
    await this.prisma.pluginProject.update({
      where: { id: pluginId },
      data: { favoriteCount: { increment: 1 } },
    });
    return { favorited: true, favoriteCount: (await this.getCount(pluginId)).favoriteCount };
  }
}
```

### 5.3 搜索与筛选 → AC-005

```typescript
// community.service.ts — search()
async search(query: SearchQuery): Promise<PaginatedResult<PluginSummary>> {
  const where: Prisma.PluginProjectWhereInput = {
    isPublished: true,
  };

  // 关键词搜索（PostgreSQL 的 insensitive contains）
  if (query.q) {
    where.OR = [
      { name: { contains: query.q, mode: 'insensitive' } },
      { description: { contains: query.q, mode: 'insensitive' } },
    ];
  }

  // 核心筛选
  if (query.coreType) {
    where.coreType = query.coreType;
  }

  // MC 版本筛选
  if (query.mcVersion) {
    where.mcVersion = query.mcVersion;
  }

  // 排序
  let orderBy: Prisma.PluginProjectOrderByWithRelationInput;
  switch (query.sort) {
    case 'popular':
      orderBy = { downloadCount: 'desc' };
      break;
    case 'rating':
      // 评分排序较复杂，MVP 阶段先用创建时间
      orderBy = { createdAt: 'desc' };
      break;
    default:  // 'latest'
      orderBy = { createdAt: 'desc' };
  }

  const [items, total] = await Promise.all([
    this.prisma.pluginProject.findMany({
      where,
      orderBy,
      skip: (query.page - 1) * query.limit,
      take: query.limit,
      include: {
        user: { select: { id: true, nickname: true, avatar: true } },
        ratings: { select: { score: true } },  // 用于前端计算均分
      },
    }),
    this.prisma.pluginProject.count({ where }),
  ]);

  // 计算均分
  const mapped = items.map(p => ({
    ...p,
    ratings: undefined,
    avgRating: p.ratings.length > 0
      ? p.ratings.reduce((sum, r) => sum + r.score, 0) / p.ratings.length
      : 0,
    ratingCount: p.ratings.length,
  }));

  return { items: mapped, total, page: query.page, limit: query.limit };
}
```

### 5.4 详情页数据聚合 → AC-006

```typescript
// community.service.ts — getDetail()
async getDetail(pluginId: string, currentUserId?: string) {
  const plugin = await this.prisma.pluginProject.findUnique({
    where: { id: pluginId, isPublished: true },
    include: {
      user: { select: { id: true, nickname: true, avatar: true, bio: true } },
      ratings: { select: { score: true } },
    },
  });

  if (!plugin) throw new NotFoundException('Plugin not found');

  // 计算均分
  const avgRating = plugin.ratings.length > 0
    ? plugin.ratings.reduce((s, r) => s + r.score, 0) / plugin.ratings.length
    : 0;

  let userState = {};
  if (currentUserId) {
    const [favorite, like, rating, follow] = await Promise.all([
      this.prisma.favorite.findUnique({ where: { userId_pluginId: { userId: currentUserId, pluginId } } }),
      this.prisma.like.findUnique({ where: { userId_targetId_targetType: { userId: currentUserId, targetId: pluginId, targetType: 'PLUGIN' } } }),
      this.prisma.rating.findUnique({ where: { userId_pluginId: { userId: currentUserId, pluginId } } }),
      this.prisma.follow.findUnique({ where: { followerId_followingId: { followerId: currentUserId, followingId: plugin.userId } } }),
    ]);

    userState = {
      isFavorited: !!favorite,
      isLiked: !!like,
      userRating: rating?.score || null,
      isFollowing: !!follow,
      isOwner: plugin.userId === currentUserId,
    };
  }

  return {
    plugin: { ...plugin, ratings: undefined, avgRating, ratingCount: plugin.ratings.length },
    ...userState,
  };
}
```

## 7. 技术决策

### 7.1 搜索用 PostgreSQL `contains` + `insensitive` 而非全文索引

**背景**：作品广场的搜索需求——用户输入关键词，匹配插件名称和描述。

**选项**：

- A: Prisma 全文搜索（`search`/`@fulltext`）— 更强大，支持排名和词干，但需要数据库 migration 创建全文索引
- B: PostgreSQL `contains` (LIKE) + `mode: 'insensitive'` — 简单，对 MVP 阶段足够，Prisma 原生支持

**结论**：选择 B。插件名称和描述通常很短（名称 < 50 字，描述 < 2000 字），`contains` 的性能完全够用。后续搜索量增长后再升级到全文搜索。

### 7.2 评分均分实时计算而非缓存

**背景**：评分均分在每次展示时需要计算。

**选项**：

- A: 实时 `_avg` 聚合查询 — 数据始终准确，每次查询需要 COUNT + AVG，PostgreSQL 在索引下很快
- B: 在 PluginProject 表缓存 `avgRating` 和 `ratingCount` 字段 — 减少查询，但评分/修改评分时需要同步更新缓存

**结论**：选择 A。作品广场的访问量在 MVP 阶段不会很高，`_avg` 聚合查询在 PostgreSQL 中执行极快（Rating 表有 `pluginId` 索引）。等到日均 PV 超过 10 万时再考虑缓存。

### 7.3 计数递增/递减在业务层而非数据库触发器

**背景**：收藏数、点赞数需要随操作更新。

**选项**：

- A: Prisma `increment` / `decrement` 在 Service 中操作 — 逻辑清晰，事务保证一致性
- B: PostgreSQL 触发器 + 函数 — 数据库层自动维护，但调试困难，NestJS 层无法感知

**结论**：选择 A。使用 Prisma 的 `update({ data: { favoriteCount: { increment: 1 } } })` 在同一个 Service 事务中完成收藏创建+计数递增，原子性由 PostgreSQL 行级锁保证。

## 8. 安全与性能

**权限检查**：

- 发布/下架/修改插件需要 JWT 认证 + 所有权检查 → 403
- 收藏/点赞/评论/评分/关注需要 JWT 认证（未登录→401）
- 浏览作品广场和详情页无需登录（`@Public()`）

**自评限制**：评分时检查 `plugin.userId !== currentUserId`，阻止自评 → AC-102

**搜索性能**：`isPublished` + `createdAt` 复合索引确保作品广场列表查询快速，`coreType` 和 `mcVersion` 的单列索引支持筛选

**数据一致性**：收藏/点赞等的切换操作在同一个 Prisma 事务中完成（`$transaction`），防止并发下的计数错乱
