# 通知 & 私信系统 — 技术设计文档

## 1. 设计概要

**功能描述**：实现站内通知系统（WebSocket 实时推送 + 通知列表 + 标记已读）和私信系统（会话管理 + 聊天面板 + WebSocket 实时消息推送），将社区的双向互动闭环打通。

**影响范围**：后端新增 `notification/` 模块；前端修改 `App.tsx`（路由）、`components/layout/TopNav.tsx`（铃铛图标+角标），新增 `pages/messages/MessagesPage.tsx`、通知弹窗组件。

**技术难点**：

- WebSocket 连接管理：需要维护 userId → socketId 的映射，支持断线重连后重新绑定
- 通知和消息的实时推送延迟控制在 1s 以内

**外部依赖**：无新增（WebSocket Gateway 已在 NestJS 中，Phase 1.2 已实现 ws-client）

## 2. 架构概览

```
通知数据流:

User A 操作 → CommunityService
                │
                ├── 执行操作（点赞/评论等）
                │
                └── NotificationService.create()
                      │
                      ├── 1. 检查不自通 (actorId !== targetId)
                      ├── 2. prisma.notification.create({ type, data })
                      ├── 3. prisma.notification.count({ where: { read: false } })
                      └── 4. NotificationGateway.push(userB, 'notification.new', { ... })
                            │
                            └── WebSocket → User B 前端
                                  ├── 右下角 Toast 弹窗
                                  ├── 铃铛角标 +1
                                  └── 通知列表刷新


私信数据流:

User A 发送消息 → MessagesService.send()
                    │
                    ├── 1. 检查不自信 (fromUserId !== toUserId)
                    ├── 2. prisma.message.create({ fromUserId, toUserId, content })
                    └── 3. NotificationGateway.push(userB, 'message.new', { ... })
                          │
                          └── WebSocket → User B 前端
                                ├── 右下角 Toast 弹窗
                                ├── 导航栏角标 +1
                                └── 如果在私信页面 → 实时插入新消息


WebSocket 连接管理:

用户登录 → ws-client.ts 连接 ws://host:3000
                │
                └── JWT 验证 → NotificationGateway.handleConnection()
                      │
                      └── Map<userId, Set<socketId>> 维护连接映射
                            │
                            └── 断线 → handleDisconnect() 移除映射
```

## 5. 核心逻辑

### 5.1 WebSocket 连接管理 → AC-002, AC-011

```typescript
// notification.gateway.ts
@WebSocketGateway({ namespace: '/ws', cors: true })
export class NotificationGateway implements OnGatewayConnection, OnGatewayDisconnect {
  // userId → Set<socketId> 映射（支持同一用户多设备/多 Tab）
  private userSockets = new Map<string, Set<string>>();

  handleConnection(client: Socket) {
    // 从 handshake query 中提取 JWT Token
    const token = client.handshake.auth?.token || client.handshake.query?.token;
    try {
      const payload = this.jwtService.verify(token);
      const userId = payload.id;

      // 绑定 userId → socketId
      if (!this.userSockets.has(userId)) {
        this.userSockets.set(userId, new Set());
      }
      this.userSockets.get(userId)!.add(client.id);

      // 加入 room（方便按 userId 广播）
      client.join(`user:${userId}`);
    } catch {
      client.disconnect(); // 无效 Token 断开连接
    }
  }

  handleDisconnect(client: Socket) {
    // 清理映射
    for (const [userId, sockets] of this.userSockets.entries()) {
      if (sockets.delete(client.id) && sockets.size === 0) {
        this.userSockets.delete(userId);
      }
    }
  }

  // 公开方法：向指定用户推送事件
  pushToUser(userId: string, event: string, data: any) {
    this.server.to(`user:${userId}`).emit(event, data);
  }
}
```

### 5.2 通知创建与推送 → AC-001, AC-002, AC-201

```typescript
// notification.service.ts
@Injectable()
export class NotificationService {
  constructor(
    private prisma: PrismaService,
    private wsGateway: NotificationGateway,
    private userService: UserService,
  ) {}

  async create(dto: CreateNotificationDto) {
    // BR-003: 不通知自己
    if (dto.userId === dto.actorId) return null; // → AC-201

    // 创建通知
    const notification = await this.prisma.notification.create({
      data: {
        userId: dto.userId,
        type: dto.type,
        data: dto.data,
      },
    });

    // 获取未读数
    const unreadCount = await this.prisma.notification.count({
      where: { userId: dto.userId, read: false },
    });

    // 通知上限检查（→ AC-104）
    const totalCount = await this.prisma.notification.count({
      where: { userId: dto.userId },
    });
    if (totalCount > 100) {
      // 删除最旧的通知
      const oldest = await this.prisma.notification.findFirst({
        where: { userId: dto.userId },
        orderBy: { createdAt: 'asc' },
      });
      if (oldest) await this.prisma.notification.delete({ where: { id: oldest.id } });
    }

    // WebSocket 推送
    this.wsGateway.pushToUser(dto.userId, 'notification.new', {
      notification,
      unreadCount,
    });

    return notification;
  }
}
```

### 5.4 私信发送 → AC-007, AC-008, AC-011, AC-101

```typescript
// message.service.ts
@Injectable()
export class MessageService {
  async send(fromUserId: string, dto: SendMessageDto) {
    // 不自信检查
    if (fromUserId === dto.toUserId) {
      throw new BadRequestException('Cannot send message to yourself'); // → AC-101
    }

    // 验证接收者存在
    const toUser = await this.prisma.user.findUnique({ where: { id: dto.toUserId } });
    if (!toUser) throw new NotFoundException('User not found');

    // 创建消息
    const message = await this.prisma.message.create({
      data: {
        fromUserId,
        toUserId: dto.toUserId,
        content: dto.content,
      },
    });

    // WebSocket 推送
    this.wsGateway.pushToUser(dto.toUserId, 'message.new', {
      message: { ...message, fromNickname: fromUser.nickname },
    });

    return message;
  }
}
```

### 5.5 会话列表查询 → AC-009

```typescript
async getConversations(userId: string) {
  // 获取所有相关的 "对方用户ID"
  const messages = await this.prisma.message.findMany({
    where: {
      OR: [{ fromUserId: userId }, { toUserId: userId }],
    },
    orderBy: { createdAt: 'desc' },
    take: 1000,  // 最多扫描 1000 条构建会话列表
  });

  // 按对方 userId 分组
  const conversationMap = new Map<string, { userId: string; lastMessage: any; unreadCount: number }>();

  for (const msg of messages) {
    const otherUserId = msg.fromUserId === userId ? msg.toUserId : msg.fromUserId;
    if (!conversationMap.has(otherUserId)) {
      const otherUser = await this.prisma.user.findUnique({
        where: { id: otherUserId },
        select: { id: true, nickname: true, avatar: true },
      });
      conversationMap.set(otherUserId, {
        userId: otherUserId,
        ...otherUser,
        lastMessage: msg,
        unreadCount: 0,
      });
    }
    // 计算未读数（对方发送的未读消息）
    if (msg.toUserId === userId && !msg.read) {
      conversationMap.get(otherUserId)!.unreadCount++;
    }
  }

  // 按最后消息时间排序
  return Array.from(conversationMap.values())
    .sort((a, b) => b.lastMessage.createdAt.getTime() - a.lastMessage.createdAt.getTime());
}
```

## 7. 技术决策

### 7.1 WebSocket 独立 Gateway 而非与 progress Gateway 合并

**背景**：同一个 NestJS 应用中，通知和编译进度的 WebSocket 放一起还是分开。

**结论**：使用独立的 `NotificationGateway`（与 `ProgressGateway` 分属不同的 Gateway 只处理各自的事件）。职责清晰，便于维护。

### 7.2 通知只存 100 条，用软删除而非硬删除

**背景**：通知存储上限和清理策略。

**选项**：

- A: 硬删除（直接 `DELETE`）— 节省存储，但删除后不可恢复
- B: 软删除（`read = true` 的不显示，但保留在数据库）— 可回溯，但数据库会持续增长

**结论**：选择创建时检查总数→超过 100 条则物理删除最旧的。通知数据的价值随时间快速衰减，100 条足够用户回溯近期的互动记录。

### 7.3 会话列表使用应用层分组而非 SQL GROUP BY

**背景**：`GET /messages/conversations` 的查询实现。

**选项**：

- A: 原始 SQL 的 `GROUP BY` + 子查询 — 性能更好，但 Prisma 不支持原生分组
- B: Prisma 查询全部相关消息 → 应用层分组 → 排序 — 代码清晰但扫描数据量较大

**结论**：选择 B。1000 条限制确保了扫描范围可控，而一般用户的消息量远低于此。应用层分组代码可读性高，易于维护。

## 8. 安全与性能

**WebSocket 安全**：

- 连接时验证 JWT Token（从 `handshake.auth.token` 或 `handshake.query.token` 提取）
- 无效 Token 直接 `client.disconnect()`
- 每次消息推送按 `user:{userId}` room 隔离，不会推送给其他用户

**消息权限**：

- 用户只能查看自己参与的消息（`fromUserId = userId OR toUserId = userId`）
- 通知只能查看和标记自己的（`userId = currentUserId`）

**性能优化**：

- 通知列表和会话列表均支持分页
- 私信历史每次加载 50 条，上拉加载更早的消息
- 通知创建和消息写入使用 `prisma.$transaction` 保证原子性
