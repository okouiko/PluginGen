# 数据库 & 公共基础设施 — 功能需求说明书

## 1. 背景与价值

**为什么做**：Phase 1.1 已完成项目骨架搭建，但项目尚不能存储任何数据或处理用户请求。数据库定义了所有业务模块的数据基石，公共基础设施（统一错误处理、认证守卫、响应格式、前端 API 客户端和布局组件）则确保后续每个 Phase 的代码都遵循一致的模式，无需重复造轮子。

**做成什么样**：后端可通过 Prisma 操作 11 张数据库表，所有 API 响应统一为 `{ code, data, message }` 格式，错误信息统一捕获并格式化，JWT 认证守卫就绪。前端可发起带 Token 的 API 请求、自动重连的 WebSocket 连接，Zustand 状态管理可用，TopNav 和 Footer 按 DESIGN.md 渲染。后续 Phase 的开发者可以直接专注写业务逻辑。

**关联**：路线图 Phase 1.2 / 里程碑一（MVP）

## 2. 范围

### 本次做

- 编写 Prisma Schema，包含 11 张核心表及其关系
- 执行 `npx prisma migrate dev` 生成初始 Migration
- 实现 Prisma 全局服务（`prisma.service.ts`）
- 实现后端公共基础设施：
  - `JwtAuthGuard` — JWT 认证守卫（保护需要登录的 API）
  - `@CurrentUser()` 装饰器 — 获取当前登录用户
  - `@Public()` 装饰器 — 标记公开 API 跳过认证
  - `AllExceptionsFilter` — 全局异常过滤器（统一错误响应格式 `{ code, message, details?, timestamp, path }`）
  - `TransformInterceptor` — 全局响应拦截器（统一成功响应格式 `{ code, data, message }`）
  - `ValidationPipe` — 全局 DTO 校验管道
  - `PaginationDto` — 共享分页 DTO
- 实现前端公共基础设施：
  - `lib/api-client.ts` — Axios 封装（基础 URL、JWT 自动注入、401 自动跳转登录）
  - `lib/ws-client.ts` — WebSocket 客户端（自动重连、事件路由）
  - `stores/auth-store.ts` — 用户认证状态管理
  - `stores/ui-store.ts` — UI 状态管理（侧栏展开等）
  - 全局路由表配置（RouterProvider + 路由常量）
  - `components/layout/TopNav.tsx` — 顶部导航栏（DESIGN.md 规范）
  - `components/layout/Footer.tsx` — 底部（DESIGN.md 规范）
  - 占位页面：登录、注册、首页（用于验证路由和布局）

### 本次不做

- Auth 的业务逻辑（注册/登录 API）— Phase 1.3
- 任何业务模块的 `service.ts` / `controller.ts` — 后续 Phase
- 数据库种子数据脚本 — 后续按需添加
- 单元测试或 e2e 测试 — 测试规范章节统一执行
- Redis 集成（Bull 队列、配额 TTL）— Phase 1.5/1.6 需要时做

## 4. 需求详述

### 4.1 核心流程

1. **开发者执行 `npx prisma migrate dev`** → Prisma 读取 `schema.prisma` → 生成 SQL Migration 文件 → 执行到 PostgreSQL → 11 张表创建完成
2. **开发者启动后端服务** → `prisma.service.ts` 在模块启动时建立数据库连接 → `GET /api/health` 返回 `{ code: 200, data: { status: "ok", database: "connected" } }`（通过 TransformInterceptor）
3. **开发者 POST 发送请求时缺少必填字段** → ValidationPipe 拦截 → 返回 `{ code: 400, message: "Validation failed", details: ["nickname should not be empty"] }`
4. **开发者访问未认证的受保护接口** → JwtAuthGuard 拦截 → 返回 `{ code: 401, message: "Unauthorized" }`
5. **前端发起带 JWT 的 API 请求** → `api-client.ts` 的 Axios 拦截器自动注入 `Authorization: Bearer xxx` → 请求到达后端 → JwtAuthGuard 验证通过 → 正常处理

### 4.2 交互规则

- 统一成功响应格式：`{ code: 200, data: <T>, message: "success" }`
- 统一错误响应格式：`{ code: <HTTP_STATUS>, message: "<错误描述>", details?: string[], timestamp: "<ISO时间>", path: "<请求路径>" }`
- 401 响应时，前端 Axios 拦截器自动清除 JWT Token 并跳转到 `/login`
- TopNav 固定在页面顶部（64px 高），Footer 固定在底部
- WebSocket 客户端自动重连策略：断开后 1s → 2s → 4s → 8s → 16s（指数退避，最大 30s）

### 4.3 业务规则

| 编号   | 规则                                                                                     |
| ------ | ---------------------------------------------------------------------------------------- |
| BR-001 | Prisma 全局服务必须作为 NestJS 单例（`@Global()`），整个应用只创建一个 PrismaClient 实例 |
| BR-002 | `@Public()` 装饰器标记的接口跳过 JWT 校验，未标记的接口默认需要认证                      |
| BR-003 | 统一响应格式必须在 TransformInterceptor 中实现，禁止在 Controller 中手动包装响应         |
| BR-004 | 所有 DTO 必须使用 class + class-validator 装饰器定义，禁止使用 interface 或 type         |
| BR-005 | 前端 API 客户端必须在 request 拦截器中注入 JWT Token，禁止在每个调用处手动添加           |
| BR-006 | 前端路由表必须集中定义在 `config/routes.ts`，禁止在页面组件中路由字面量                  |

## 5. 验收标准

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001**: Prisma Schema 用户表
  - **Given** Prisma Schema 已定义
  - **When** 查看 `schema.prisma` 的 `User` 模型
  - **Then** 包含字段：`id` (UUID)、`email` (String, unique)、`password` (String)、`nickname` (String)、`avatar` (String, 默认 URL)、`bio` (String, 可选)、`level` (Int, 默认 1)、`exp` (Int, 默认 0)、`dailyQuota` (Int, 默认 20)、`createdAt`、`updatedAt`

- [ ] **AC-002**: Prisma Schema 插件项目表
  - **Given** Prisma Schema 已定义
  - **When** 查看 `PluginProject` 模型
  - **Then** 包含字段：`id` (UUID)、`userId` (关联 User)、`name`、`description`、`mcVersion` (String)、`coreType` (CoreType enum)、`javaVersion` (String)、`packageName`、`status` (ProjectStatus enum)、`currentVersion` (Int)、`isPublished` (Boolean, 默认 false)、`downloadCount` (Int, 默认 0)、`favoriteCount` (Int, 默认 0)、`starCount` (Int, 默认 0)、`createdAt`、`updatedAt`

- [ ] **AC-003**: Prisma Schema 核心类型枚举
  - **Given** Prisma Schema 已定义
  - **When** 查看 `CoreType` 枚举
  - **Then** 包含值：`BUKKIT`、`SPIGOT`、`PAPER`、`PURPUR`、`BUNGEECORD`、`VELOCITY`

- [ ] **AC-004**: Prisma Schema 版本表
  - **Given** Prisma Schema 已定义
  - **When** 查看 `PluginVersion` 模型
  - **Then** 包含字段：`id`、`pluginId` (关联 PluginProject)、`version` (Int)、`filesManifest` (Json)、`compileStatus` (CompileStatus enum)、`compileLog` (Text, 可选)、`createdAt`

- [ ] **AC-005**: Prisma Schema 社区表
  - **Given** Prisma Schema 已定义
  - **When** 查看社区相关模型
  - **Then** 包含 `Favorite`（userId + pluginId, unique pair）、`Comment`（userId + pluginId + content）、`Rating`（userId + pluginId + score 1-5, unique pair）、`Like`（userId + targetId + targetType）、`Follow`（followerId + followingId, unique pair）

- [ ] **AC-006**: Prisma Schema 通知与签到表
  - **Given** Prisma Schema 已定义
  - **When** 查看通知和签到模型
  - **Then** 包含 `Notification`（userId + type + data Json + read Boolean）、`Message`（fromUserId + toUserId + content + read Boolean）、`DailyCheckIn`（userId + date + streak Int）、`DailyTask`（userId + date + taskType + progress Int + completed Boolean）

- [ ] **AC-007**: 统一成功响应格式
  - **Given** TransformInterceptor 已注册
  - **When** 调用 `GET /api/health`
  - **Then** 返回 JSON：`{ "code": 200, "data": { "status": "ok", "database": "connected" }, "message": "success" }`

- [ ] **AC-008**: 统一错误响应格式
  - **Given** AllExceptionsFilter 已注册
  - **When** 调用一个不存在的 API（如 `GET /api/nonexistent`）
  - **Then** 返回 JSON：`{ "code": 404, "message": "Cannot GET /api/nonexistent", "timestamp": "<ISO 时间>", "path": "/api/nonexistent" }`

- [ ] **AC-009**: JWT 守卫拦截未认证请求
  - **Given** 某个 Controller 上标记了 `@UseGuards(JwtAuthGuard)`（未标记 `@Public()`）
  - **When** 不携带 `Authorization` 头发起请求
  - **Then** 返回 401 + `{ "code": 401, "message": "Unauthorized" }`

- [ ] **AC-010**: API 客户端自动注入 Token
  - **Given** 前端 `auth-store.ts` 中存储了有效的 JWT Token
  - **When** 通过 `api-client.ts` 发起任意 API 请求
  - **Then** 请求头包含 `Authorization: Bearer <token>`（通过 Axios request 拦截器自动注入）

- [ ] **AC-011**: TopNav 渲染符合 DESIGN.md
  - **Given** 浏览器打开前端应用
  - **When** 查看页面顶部
  - **Then** 显示 TopNav：高度 64px，背景色 `#faf9f5`（`bg-canvas`），左侧显示品牌名称，右侧包含"登录"文字链接（珊瑚色 `#cc785c`）

- [ ] **AC-012**: Footer 渲染符合 DESIGN.md
  - **Given** 浏览器打开前端应用
  - **When** 滚动到页面底部
  - **Then** 显示 Footer：背景色 `#181715`（`bg-surface-dark`），文字颜色 `#a09d96`（`text-on-dark-soft`），包含版权信息和项目链接

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-101**: DTO 校验失败
  - **Given** DTO 中标记了 `@IsEmail()` 的字段传入了非法邮箱格式
  - **When** 调用对应 API
  - **Then** 返回 400 + `{ "code": 400, "message": "Validation failed", "details": ["email must be an email"] }`

- [ ] **AC-102**: 401 Token 过期
  - **Given** JWT Token 已过期（超过 7 天）
  - **When** 携带过期 Token 访问受保护 API
  - **Then** 返回 401 + `{ "code": 401, "message": "Token expired" }`，前端 Axios 拦截器清除本地 Token 并跳转到 `/login`

- [ ] **AC-103**: 后端数据库连接断开
  - **Given** PostgreSQL 还未启动或已断开
  - **When** 调用 `GET /api/health`
  - **Then** 响应中 `code: 200`、`data.status: "error"`、`data.database: "disconnected"`（PrismaService 优雅降级，不崩溃）

- [ ] **AC-104**: WebSocket 自动重连
  - **Given** WebSocket 客户端已连接但服务器断开
  - **When** 服务器恢复可用
  - **Then** WebSocket 客户端在指数退避（1s → 2s → 4s → ... → 最大 30s）后自动重新连接成功

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-201**: PrismaService 全局单例 ← BR-001
  - **Given** `prisma.service.ts` 在模块声明中标记为 `@Global()`
  - **When** 在任意 Service 中注入 `PrismaService`
  - **Then** 注入的是同一个单例实例，不会创建重复数据库连接

- [ ] **AC-202**: `@Public()` 跳过认证 ← BR-002
  - **Given** Controller 上标记了 `@UseGuards(JwtAuthGuard)`，健康检查路由标记了 `@Public()`
  - **When** 不携带 Token 访问 `/api/health`
  - **Then** 请求正常通过，返回健康检查数据（不返回 401）

- [ ] **AC-203**: TransformInterceptor 统一响应 ← BR-003
  - **Given** 任意 Controller 返回 `return { data: "test" }`
  - **When** 调用该 API
  - **Then** 响应体被自动包裹为 `{ "code": 200, "data": { "data": "test" }, "message": "success" }`

- [ ] **AC-204**: 路由表集中定义 ← BR-006
  - **Given** `config/routes.ts` 文件
  - **When** 查看其导出
  - **Then** 定义了所有页面路由路径常量（如 `ROUTES.HOME = '/'`、`ROUTES.LOGIN = '/login'`、`ROUTES.REGISTER = '/register'`、`ROUTES.DASHBOARD = '/dashboard'` 等），页面组件使用常量而非字符串字面量
