# 作品广场 — 功能需求说明书

## 1. 背景与价值

**为什么做**：经过 MVP 的 7 个 Phase，用户已经可以完成「AI 生成→编译→下载」的个人闭环。但一个平台的价值不仅在于工具本身，更在于用户之间的连接——作品广场让插件从「私人物品」变成「社区公共作品」，用户可以发现别人的插件获取灵感，也可以分享自己的作品获得认可。收藏、点赞、评论、评分构成了社区的互动飞轮，是平台从工具进化为社区的关键转折点。

**做成什么样**：用户进入作品广场 → 看到瀑布流/网格展示的公开插件 → 可以按核心类型、MC 版本、标签搜索和筛选 → 点击进入插件详情页 → 查看完整介绍、截图、下载量/评分/评论 → 可收藏、点赞、评论、评分 → 关注创作者 → 回到个人主页看到关注列表。

**关联**：路线图 Phase 2.1 / 里程碑二（功能完整版）

## 2. 范围

### 本次做

- 后端：插件发布/下架（`POST /plugins/:id/publish`、`POST /plugins/:id/unpublish`）
- 后端：作品广场列表（`GET /community/plugins` — 分页 + 按核心/MC 版本/热度/时间排序 + 全文搜索）
- 后端：收藏功能（`POST /community/favorite` — 切换收藏/取消收藏）
- 后端：点赞功能（`POST /community/like` — 切换点赞/取消点赞）
- 后端：评论功能（`POST /community/comment` + `GET /community/plugins/:id/comments`）
- 后端：评分功能（`POST /community/rating` + `GET /community/plugins/:id/rating` — 1-5 星）
- 后端：关注功能（`POST /community/follow` — 切换关注/取关 + `GET /community/followers`）
- 后端：作品广场统计计数（暴露在 API 中：下载量、收藏数、点赞数、评分均分、评论数）
- 前端：作品广场列表页（`/workspace` — 网格布局 + 搜索框 + 筛选标签栏 + 排序）
- 前端：插件详情页（`/workspace/:id` — 介绍/统计/评论/评分/收藏按钮/下载入口）
- 前端：收藏/点赞/评论/评分的交互组件

### 本次不做

- 作品推荐算法（按用户偏好推荐）— 后续迭代
- 作品审核机制 — 已确认直接发布，无审核
- 作品分类标签管理（标签由 AI 生成时自动提取）— 后续迭代
- 作品截图/展示图上传 — 后续迭代
- 作品置顶功能 — 创作者体系做到
- 举报/屏蔽功能 — 后续迭代
- 评论的回复（嵌套评论）— 后续迭代

## 4. 需求详述

### 4.1 核心流程

#### 流程 1：发布插件

1. 用户在编辑页完成编译并确认插件可用
2. 用户点击"发布到作品广场"按钮
3. 弹窗让用户填写或确认：插件名称、简短介绍、MC 版本、核心类型（自动填入）、标签（可选，如"经济"、"管理"、"娱乐"）
4. 用户点击"发布" → 插件状态变为已发布，出现在作品广场
5. 创作者获得 +20 经验值
6. 关注该创作者的粉丝收到通知

#### 流程 2：浏览与搜索

1. 用户访问 `/workspace` → 展示作品广场列表
2. 默认按最新发布排序，每页 20 个
3. 用户可以按核心类型（Bukkit/Spigot/Paper 等）筛选
4. 用户可以在搜索框输入关键词搜索插件名称和描述
5. 搜索结果显示匹配的插件卡片，包含名称、作者、核心类型标签、评分、下载量

#### 流程 3：插件详情互动

1. 用户点击插件卡片 → 进入 `/workspace/:id` 详情页
2. 详情页展示：插件名称、作者头像+昵称、核心类型标签、评分星星、下载量/收藏量/点赞数、完整介绍
3. 用户可操作：收藏（切换）、点赞（切换）、评分（1-5 星）、写评论
4. 底部展示评论区：按时间倒序展示所有评论
5. 右侧或底部显示下载按钮（源码 ZIP + 编译 JAR）
6. 用户可点击作者头像进入其个人主页

#### 流程 4：关注创作者

1. 在插件详情页或个人主页点击"关注"按钮
2. 关注后按钮变为"已关注"
3. 再次点击取消关注
4. 被关注的创作者获得通知

### 4.2 交互规则

- 发布按钮仅在编译成功后可用（状态为 COMPILED 或已发布过的重新修改后）
- 收藏/点赞/评论/评分均需登录，未登录点击时弹出登录提示或跳转登录页
- 收藏和评分每个用户对每个插件只能操作一次（收藏可切换，评分可修改）
- 评论按时间倒序排列，不支持嵌套回复
- 作品广场默认按最新发布排序，可选按热度（下载量）排序
- 搜索输入防抖 300ms，输入时实时搜索
- 关注按钮在个人主页和插件详情页的作者区域都有

### 4.3 业务规则

| 编号   | 规则                                                                         |
| ------ | ---------------------------------------------------------------------------- |
| BR-001 | 发布后的插件在编辑页修改后不会自动更新到作品广场——需要用户手动"更新发布版本" |
| BR-002 | 评分 1-5 星，整数分，每个用户对每个插件只有一个评分，可修改                  |
| BR-003 | 收藏/点赞/评论/关注均为切换操作（同一点击=创建，再次点击=取消）              |
| BR-004 | 已下架的插件在作品广场不再展示，但已收藏/已评论的用户仍可在个人页面看到      |
| BR-005 | 创作者不能给自己的插件评分，但可以评论                                       |

## 5. 验收标准

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001**: 发布插件到作品广场
  - **Given** 用户已完成编译（状态为 COMPILED）
  - **When** 用户点击"发布到作品广场"，填写介绍并确认
  - **Then** 插件状态变为 PUBLISHED，出现在作品广场列表中，创作者经验值 +20

- [ ] **AC-002**: 下架插件
  - **Given** 插件已发布
  - **When** 用户点击"下架"
  - **Then** 插件从作品广场列表中消失

- [ ] **AC-003**: 作品广场列表展示
  - **Given** 有多个已发布的插件
  - **When** 访问 `/workspace`
  - **Then** 以网格形式展示插件卡片，每张卡片包含插件名称、作者昵称、核心类型标签、评分均分、下载量

- [ ] **AC-004**: 作品广场分页
  - **Given** 已发布的插件超过 20 个
  - **When** 浏览作品广场
  - **Then** 每页显示 20 个，底部有分页按钮（"上一页"/"下一页"）

- [ ] **AC-005**: 搜索与筛选
  - **Given** 作品广场有多个插件
  - **When** 用户在搜索框输入"签到"并选择筛选条件"Paper"
  - **Then** 列表仅显示名称或描述中包含"签到"且核心类型为 Paper 的插件

- [ ] **AC-006**: 插件详情页
  - **Given** 用户点击某个插件卡片
  - **When** 进入详情页
  - **Then** 显示完整信息：插件名称、作者（可点击到个人主页）、核心类型、MC 版本、评分（均分+投票数）、下载量/收藏量/点赞数、介绍文字、评论区、下载按钮

- [ ] **AC-007**: 收藏插件
  - **Given** 用户已登录且在插件详情页
  - **When** 用户点击收藏图标
  - **Then** 图标变为实心已收藏状态，收藏数 +1
  - **When** 再次点击
  - **Then** 图标恢复为空心未收藏状态，收藏数 -1

- [ ] **AC-008**: 点赞插件
  - **Given** 用户已登录且在插件详情页
  - **When** 点击点赞按钮
  - **Then** 按钮变为已点赞状态，点赞数 +1
  - **When** 再次点击
  - **Then** 取消点赞，点赞数 -1

- [ ] **AC-009**: 评论插件
  - **Given** 用户已登录且在插件详情页
  - **When** 输入评论内容（如"很好用的签到插件！"）并提交
  - **Then** 评论出现在评论区最上方，显示用户昵称、头像、评论时间和内容

- [ ] **AC-010**: 评分插件
  - **Given** 用户已登录且在插件详情页
  - **When** 点击 4 颗星提交评分
  - **Then** 评分成功，插件评分均分更新为该用户新评分参与计算后的值
  - **When** 用户再次点击 5 颗星
  - **Then** 评分更新为 5 星，均分重新计算

- [ ] **AC-011**: 从详情页下载
  - **Given** 插件已发布且有编译成功的历史版本
  - **When** 任何用户（包括未登录）在详情页点击"下载 JAR"
  - **Then** 下载对应的 `.jar` 文件，下载量 +1

- [ ] **AC-012**: 关注创作者
  - **Given** 用户已登录
  - **When** 在插件详情页点击作者旁的"关注"按钮
  - **Then** 按钮变为"已关注"，关注列表增加该创作者

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-101**: 未登录点击收藏/点赞/评论/评分
  - **Given** 用户未登录
  - **When** 点击收藏/点赞/评论/评分按钮
  - **Then** 弹出提示"请先登录"或自动跳转到登录页

- [ ] **AC-102**: 创作者给自己的插件评分
  - **Given** 插件作者本人查看自己的插件详情页
  - **When** 查看评分区域
  - **Then** 评分组件置灰，提示"不能给自己的插件评分"

- [ ] **AC-103**: 插件尚未编译完成时发布
  - **Given** 插件状态为 DRAFT 或 COMPILING
  - **When** 用户尝试点击"发布"
  - **Then** 按钮置灰并提示"请先完成编译"

- [ ] **AC-104**: 搜索无结果
  - **Given** 作品广场中没有匹配的插件
  - **When** 用户输入不存在的关键词搜索
  - **Then** 显示"没有找到匹配的插件"空状态提示

- [ ] **AC-105**: 收藏/点赞/评论后数据立即可见
  - **Given** 用户 A 对插件进行了收藏/点赞/评论
  - **When** 用户 B 刷新插件详情页
  - **Then** 用户 B 能看到用户 A 的评论以及更新后的计数

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-201**: 发布后修改需手动更新 ← BR-001
  - **Given** 插件已发布
  - **When** 作者在编辑器修改代码并重新编译成功
  - **Then** 作品广场仍显示旧版本，作者需点击"更新发布版本"来同步

- [ ] **AC-202**: 评分均分计算正确 ← BR-002
  - **Given** 插件有 4 个评分（5, 4, 5, 4）
  - **When** 查询插件详情
  - **Then** 均分显示为 4.5，评分数显示为 4

- [ ] **AC-203**: 已下架不展示 ← BR-004
  - **Given** 某插件已下架
  - **When** 搜索或浏览作品广场
  - **Then** 该插件不出现在列表中
