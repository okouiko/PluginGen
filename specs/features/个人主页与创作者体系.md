# 个人主页 & 创作者体系 — 功能需求说明书

## 1. 背景与价值

**为什么做**：作品广场和通知系统让插件有了"公共"和"互动"的属性，但每个创作者的「身份」仍然是割裂的——用户不知道这个插件是谁做的、他还有什么作品、他的水平如何。个人主页将创作者的身份聚合在一起：展示个人信息、所有作品、数据统计和等级，让「关注一个人」变得有意义。创作者等级体系则提供了持续的激励机制——生成更多插件、获得更多认可，等级和称号随之提升，形成一个正向循环。

**做成什么样**：用户点击任何地方出现的作者头像/昵称 → 进入他的个人主页 → 看到头像、昵称、简介、等级徽章、经验条、数据统计（总下载量/作品数/获赞数/粉丝数）→ 下面展示他发布的所有插件（可置顶）→ 可以关注他 → 创作者自己看到等级提升有成就感。

**关联**：路线图 Phase 2.3 / 里程碑二（功能完整版）

## 2. 范围

### 本次做

- 后端：个人主页聚合 API（`GET /user/profile/:id` — 用户信息 + 作品列表 + 数据统计）
- 后端：置顶作品（`PATCH /user/pinned-plugins` — 设置最多 3 个置顶作品）
- 后端：经验值事件钩子（生成插件 +10、发布 +20、获赞 +5、被收藏 +3、签到 +5）
- 后端：等级计算公式（1-50 级，每级所需经验 = level × 100）
- 后端：等级称号（1-5 级"新手创客"、6-15 级"进阶工匠"、16-30 级"资深开发者"、31-50 级"大师"）
- 前端：个人主页页面（`/user/:id` — 公开可访问）
- 前端：等级徽章组件（`LevelBadge.tsx` — 显示等级数字 + 称号）
- 前端：经验条组件（当前经验 / 升级所需经验）
- 前端：统计卡片组件（`UserStats.tsx` — 作品数/下载量/获赞数/粉丝数）
- 前端：作品列表区域（展示该用户所有已发布的作品）
- 前端：置顶作品展示（在作品列表顶部展示置顶的 3 个）
- 前端：关注按钮（在个人主页头部）

### 本次不做

- 创作者等级特权（如每日额外配额、特殊徽章样式）— 后续迭代
- 创作者排行榜 — 后续迭代
- 个人主页自定义背景/主题 — 后续迭代
- 作品集按分类/时间筛选 — 后续迭代
- 成就系统（如"发布第一个插件"等里程碑成就）— 后续迭代

## 4. 需求详述

### 4.1 核心流程

#### 流程 1：查看个人主页

1. 用户点击任意位置的作者头像或昵称（作品广场卡片、插件详情页、评论头像等）
2. 进入 `/user/:id` 页面
3. 页面头部展示：
   - 头像（大尺寸）
   - 昵称
   - 等级徽章 + 称号（如 "Lv.12 进阶工匠"）
   - 经验条（当前经验 / 下一级所需经验）
   - 个人简介
   - 关注按钮（如果是其他用户）或"编辑资料"按钮（如果是自己）
4. 统计卡片区：4 个数字（作品数 / 总下载量 / 获赞数 / 粉丝数）
5. 作品列表：展示该用户所有已发布的作品，按最新发布排序

#### 流程 2：置顶作品

1. 创作者在自己的个人主页点击"管理置顶"
2. 弹窗展示他的所有作品列表
3. 用户选择最多 3 个作品作为置顶
4. 保存后，置顶作品出现在作品列表顶部（带"置顶"标记）
5. 其他用户访问个人主页时，置顶作品始终在前

#### 流程 3：等级成长

1. 用户每次触发经验值事件（生成插件、发布、获赞等），系统自动加分
2. 当经验值达到升级阈值时，等级自动提升
3. 提升等级时显示升级提示弹窗（"恭喜！你已升级为 Lv.3 进阶工匠！"）
4. 等级提升无上限（最高 50 级），经验值持续累积

### 4.2 交互规则

- 个人主页所有信息对所有用户（包括未登录）可见
- 编辑资料按钮仅在当前用户访问自己的主页时显示
- 关注按钮在其他用户的个人主页显示
- 置顶管理仅对自己可见
- 等级称号在 Lv.1-5 显示"新手创客"，Lv.6-15"进阶工匠"，Lv.16-30"资深开发者"，Lv.31-50"大师"
- 经验条显示格式："当前经验 / 下一级所需经验"（如 "450 / 1200"）
- 统计数字带单位说明，并排展示
- 作品列表中的每个作品显示：名称、核心类型标签、评分、下载量

### 4.3 业务规则

| 编号   | 规则                                                                                        |
| ------ | ------------------------------------------------------------------------------------------- |
| BR-001 | 等级 1-50 级，每级所需经验 = level × 100（1 级升 2 级需 100 exp，10 级升 11 级需 1000 exp） |
| BR-002 | 经验值来源：生成插件 +10、发布 +20、获赞 +5、被收藏 +3、签到 +5                             |
| BR-003 | 置顶作品最多 3 个，超出时提示取消一个                                                       |
| BR-004 | 未发布的作品不在个人主页展示                                                                |
| BR-005 | 统计数字实时更新（经验值、等级、下载量、获赞数等随操作即时变化）                            |

## 5. 验收标准

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001**: 个人主页头部展示
  - **Given** 访问 `/user/:id`
  - **When** 页面加载完成
  - **Then** 显示用户头像（大尺寸）、昵称、等级徽章 + 称号（如 "Lv.1 新手创客"）、经验条、个人简介

- [ ] **AC-002**: 统计卡片展示
  - **Given** 在个人主页
  - **When** 查看统计区域
  - **Then** 并排展示 4 个统计数字：作品数、总下载量、获赞数、粉丝数，每个数字带单位标签

- [ ] **AC-003**: 作品列表展示
  - **Given** 用户有已发布的作品
  - **When** 查看个人主页的作品列表区域
  - **Then** 按最新发布时间倒序展示所有作品，每个作品显示名称、核心类型标签、评分、下载量

- [ ] **AC-004**: 查看自己的个人主页
  - **Given** 当前用户查看自己的个人主页
  - **When** 页面加载
  - **Then** 头部显示"编辑资料"按钮（非"关注"按钮），可点击进入资料编辑

- [ ] **AC-005**: 置顶作品
  - **Given** 用户在自己的个人主页
  - **When** 设置 2 个置顶作品
  - **Then** 置顶作品始终显示在作品列表顶部，带有"📌 置顶"标记，其余作品按时间排序在后面

- [ ] **AC-006**: 等级提升触发
  - **Given** 用户经验值为 95，下一级需 100 exp
  - **When** 用户签到（+5 exp）
  - **Then** 经验值变为 100，等级从 1 升到 2，称号从"新手创客"变为"进阶工匠"，页面弹出升级庆祝提示

- [ ] **AC-007**: 经验条显示
  - **Given** 用户 Lv.10，当前经验 750
  - **When** 查看个人主页
  - **Then** 经验条显示 "750 / 1000"（下一级需 1000 exp），进度条填充 75%

- [ ] **AC-008**: 个人主页关注按钮
  - **Given** 用户 A 访问用户 B 的个人主页（A ≠ B）
  - **When** 点击"关注"按钮
  - **Then** 按钮变为"已关注"，粉丝数 +1
  - **When** 再次点击
  - **Then** 取消关注，粉丝数 -1

- [ ] **AC-009**: 生成插件 +10 exp
  - **Given** 用户当前 exp = 0
  - **When** AI 生成一个插件
  - **Then** 用户 exp = 10

- [ ] **AC-010**: 发布作品 +20 exp
  - **Given** 用户当前 exp = 10
  - **When** 用户发布一个插件到作品广场
  - **Then** 用户 exp = 30

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-101**: 访问不存在的用户
  - **Given** 用户 ID 不存在
  - **When** 访问 `/user/:id`
  - **Then** 显示"用户不存在"提示

- [ ] **AC-102**: 未发布作品的用户
  - **Given** 新注册用户，还没有发布任何作品
  - **When** 查看他的个人主页
  - **Then** 作品列表区域显示"暂无作品"空状态

- [ ] **AC-103**: 取消置顶
  - **Given** 用户有 3 个置顶作品
  - **When** 取消某个作品的置顶
  - **Then** 该作品回到正常排序位置，置顶数变为 2

- [ ] **AC-104**: 置顶超过 3 个
  - **Given** 用户已有 3 个置顶作品
  - **When** 尝试再置顶第 4 个
  - **Then** 提示"最多置顶 3 个作品，请先取消一个"

- [ ] **AC-105**: 用户无个人简介
  - **Given** 用户没有填写个人简介
  - **When** 查看个人主页
  - **Then** 简介区域显示"这个人很懒，什么都没有留下"占位文字

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-201**: 等级计算公式 ← BR-001
  - **Given** 用户 Lv.5
  - **When** 计算升到 Lv.6 所需经验
  - **Then** 所需经验 = 5 × 100 = 500

- [ ] **AC-202**: 经验值来源验证 ← BR-002
  - **Given** 用户触发不同行为
  - **When** 检查经验值变化
  - **Then** 生成 +10，发布 +20，获赞 +5，被收藏 +3，签到 +5

- [ ] **AC-203**: 未发布作品不展示 ← BR-004
  - **Given** 用户有草稿状态和已发布状态的插件
  - **When** 查看个人主页作品列表
  - **Then** 只展示已发布的插件，不展示草稿

- [ ] **AC-204**: 统计数字实时更新 ← BR-005
  - **Given** 某个用户有一个插件
  - **When** 该插件的下载量增加
  - **Then** 刷新个人主页后，总下载量统计同步更新
