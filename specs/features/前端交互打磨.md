# 前端交互打磨 — 功能需求说明书

## 1. 背景与价值

**为什么做**：经过 MVP 和完整版的 11 个 Phase，PluginGen 的"功能完整性"已经达标，但"体验细腻度"还有欠缺。当前前端在桌面宽屏下表现良好，但移动端布局尚未适配；页面切换缺少过渡动画，显得生硬；缺少暗色模式，用户在不同光线条件下使用体验不佳。这些打磨工作虽然不影响功能可用性，但直接影响用户对平台的"品质感"判断——一个交互流畅、视觉统一的产品会让用户更愿意持续使用。

**做成什么样**：用户在手机浏览器上打开 PluginGen → 导航栏自动折叠为汉堡菜单 → 页面布局自适应单列 → 代码窗口可横向滚动。用户切换到暗色模式 → 所有页面自动切换为 DESIGN.md 定义的深色面配色。页面切换时有平滑过渡动画 → 操作的反馈更加生动。键盘用户可以通过 Tab 键顺畅导航所有交互元素。

**关联**：路线图 Phase 3.1 / 里程碑三（增强版）

## 2. 范围

### 本次做

- 响应式布局适配（768px/1024px 两个主要断点）
  - 导航栏：768px 以下折叠为汉堡菜单 + 全屏 Drawer
  - 首页：Hero 标题 64px → 32px，Feature Cards 3→1 列
  - 编辑页三栏：三栏 → 上下堆叠（文件树/编辑器/版本历史通过 Tab 切换）
  - 作品广场：网格 4→2→1 列
  - 私信页面：左右分栏 → 全屏聊天（会话列表通过返回按钮切换）
  - 所有表格和列表在小屏下支持横向滚动
- 暗色/亮色模式切换
  - 基于 DESIGN.md 已定义的暗色面 token（surface-dark 等）
  - 导航栏切换按钮（太阳/月亮图标）
  - 用户偏好持久化到 localStorage
  - 跟随系统主题（`prefers-color-scheme`）作为默认值
- 页面过渡动画（Framer Motion 轻量入场动画）
  - 页面切换：淡入 + 轻微上移（duration 0.3s）
  - 列表项入场：逐个淡入（stagger 0.05s）
  - 弹窗/对话框：缩放淡入（scale 0.95 → 1.0）
  - Toast 通知：从右滑入
- 全局无障碍
  - 所有可交互元素添加 `aria-label`
  - 表单输入框关联 `aria-describedby` 错误提示
  - 图片添加 `alt` 文本
  - 键盘 Tab 导航顺序合理（可见 focus 样式）
  - 颜色对比度符合 WCAG AA 标准（4.5:1）
- 导航栏汉堡菜单（移动端）
  - 点击汉堡图标 → 全屏奶油色 Drawer 展开
  - 包含所有导航链接 + 用户菜单（如已登录）
  - 点击链接后自动关闭
  - 点击遮罩层或按 Esc 关闭

### 本次不做

- 深色模式下的代码编辑器主题切换 — 后续迭代
- 自定义主题色配置 — 后续迭代
- 触控手势支持（滑动返回等）— 后续迭代
- 动画开关（用户关闭动画选项）— 后续迭代
- PWA / 离线缓存 — 后续迭代
- 性能优化（代码分割、懒加载）— Phase 3.2
- 骨架屏优化 — 已在 Phase 1.7 实现

## 4. 需求详述

### 4.1 核心流程

#### 流程 1：移动端浏览

1. 用户在手机浏览器打开 PluginGen
2. 导航栏自动折叠，左侧显示品牌名，右侧显示汉堡菜单图标
3. 用户点击汉堡图标 → 全屏 Drawer 从右侧滑入，显示所有导航链接
4. 用户点击某个链接 → Drawer 关闭，页面跳转
5. 首页 Feature Cards 从三列变为单列堆叠
6. 作品广场网格自适应为单列
7. 代码编辑器可在小屏下水平滚动

#### 流程 2：暗色模式切换

1. 用户点击导航栏的太阳/月亮图标
2. 整个页面无缝切换为暗色配色
3. 所有组件：背景变为 `surface-dark` 色系，文字变为 `on-dark` 色系
4. 用户偏好保存到 localStorage
5. 下次访问时自动应用
6. 系统主题变更时（如 macOS 自动切换）跟随系统设置

#### 流程 3：页面过渡动画

1. 用户点击导航链接 → 当前页面淡出（轻微下移）
2. 新页面淡入（轻微上移），总时长 0.3s
3. 列表页（如作品广场）的卡片逐个淡入
4. 弹窗打开时从中心缩放淡入

### 4.2 交互规则

- 汉堡菜单的 Drawer 覆盖全屏，背景奶油色，点击链接或外部遮罩层关闭
- 暗色模式切换时使用 CSS `transition` 实现平滑过渡（0.2s）
- 动画使用 Framer Motion 的 `AnimatePresence` 实现进入/离开动画
- 移动端编辑页（三栏布局）改为 Tab 切换：文件树/编辑器/版本历史
- 代码窗口/表格在移动端支持水平滚动（`overflow-x-auto`）
- focus 样式：使用 DESIGN.md 的 `text-input-focused` 规范（珊瑚色 3px 外环）
- 所有动画遵循系统 `prefers-reduced-motion`，如用户开启减少动效则关闭动画

### 4.3 业务规则

| 编号   | 规则                                                                  |
| ------ | --------------------------------------------------------------------- | ------ | --------- |
| BR-001 | 断点 768px（移动端）/ 1024px（平板）/ 1440px（宽屏）                  |
| BR-002 | 暗色模式切换后所有页面一致生效，不包括 Monaco Editor 主题（后续迭代） |
| BR-003 | 汉堡菜单只在 < 768px 时显示，≥ 768px 显示完整导航栏                   |
| BR-004 | 用户偏好存储：`localStorage.theme = 'light'                           | 'dark' | 'system'` |
| BR-005 | 动画时长 ≤ 0.3s，不影响操作响应速度                                   |

## 5. 验收标准

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001**: 移动端导航栏折叠
  - **Given** 浏览器宽度 < 768px
  - **When** 访问任意页面
  - **Then** 导航栏仅显示品牌名和汉堡菜单图标，不显示完整导航链接

- [ ] **AC-002**: 汉堡菜单展开与关闭
  - **Given** 浏览器宽度 < 768px
  - **When** 点击汉堡图标
  - **Then** 全屏 Drawer 从右侧滑入，显示所有导航链接 + 用户菜单
  - **When** 点击任一导航链接
  - **Then** Drawer 关闭，页面跳转到对应页面

- [ ] **AC-003**: 移动端首页布局自适应
  - **Given** 浏览器宽度 < 768px
  - **When** 访问首页
  - **Then** Hero 标题字号缩小为 32px，Feature Cards 从三列变为单列堆叠，代码窗口可水平滚动

- [ ] **AC-004**: 暗色模式切换
  - **Given** 用户当前使用亮色模式
  - **When** 点击导航栏的月亮图标
  - **Then** 页面切换为暗色模式：背景变为 `#181715`（surface-dark），文字变为 `#faf9f5`（on-dark），主按钮仍为珊瑚色

- [ ] **AC-005**: 暗色模式持久化
  - **Given** 用户切换为暗色模式
  - **When** 刷新页面或关闭浏览器重新打开
  - **Then** 页面仍为暗色模式（偏好已保存到 localStorage）

- [ ] **AC-006**: 页面过渡动画
  - **Given** 用户在任意页面
  - **When** 点击导航链接切换到另一个页面
  - **Then** 当前页面淡出，新页面在 0.3s 内淡入，过程中无白屏闪烁

- [ ] **AC-007**: 键盘导航
  - **Given** 用户在任意页面
  - **When** 连续按 Tab 键
  - **Then** focus 按合理顺序移动（导航栏 → 主内容区 → 底部），每个可交互元素都有可见的 focus 样式（珊瑚色外环）

- [ ] **AC-008**: 操作元素有 aria-label
  - **Given** 页面上的按钮、链接、输入框等交互元素
  - **When** 检查 HTML
  - **Then** 每个交互元素都有 `aria-label` 或关联的文本标签（如 "关闭"、"搜索"、"签到"等）

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-101**: 系统主题跟随
  - **Given** 用户设置的主题偏好为"跟随系统"
  - **When** 操作系统从亮色切换到暗色模式
  - **Then** PluginGen 自动切换为暗色模式，无需用户手动操作

- [ ] **AC-102**: 减少动效偏好
  - **Given** 用户在系统设置中开启了"减少动效"（prefers-reduced-motion）
  - **When** 页面切换
  - **Then** 过渡动画被禁用，页面直接切换（无淡入淡出）

- [ ] **AC-103**: 平板端布局
  - **Given** 浏览器宽度 768-1024px
  - **When** 访问作品广场
  - **Then** 导航栏完整显示，Feature Cards 2 列，作品广场网格 2 列

- [ ] **AC-104**: 颜色对比度
  - **Given** 所有页面
  - **When** 检查文字和背景的对比度
  - **Then** 正文（#3d3d3a）在奶油色背景（#faf9f5）上对比度 ≥ 7:1，小字（#6c6a64）在奶油色背景上 ≥ 4.5:1

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-201**: 汉堡菜单响应式条件 ← BR-003
  - **Given** 浏览器窗口从 800px 缩小到 700px
  - **When** 导航栏变化
  - **Then** ≥ 768px 显示完整导航，< 768px 自动折叠为汉堡菜单

- [ ] **AC-202**: 暗色模式全局生效 ← BR-002
  - **Given** 用户在亮色模式下切换为暗色
  - **When** 检查所有页面
  - **Then** 首页、工作台、编辑页、作品广场、个人主页、私信页、签到页均为暗色模式
