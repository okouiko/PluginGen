# 个人主页 & 创作者体系 — 技术设计文档

## 1. 设计概要

**功能描述**：实现公开的个人主页（聚合用户信息、数据统计、作品列表、置顶作品、关注按钮）和创作者成长体系（5 种经验值事件、50 级阶梯公式、等级称号）。

**影响范围**：后端修改 `user/` 模块（新增 profile aggregator）；前端新增 `pages/user/Profile.tsx`、`components/user/LevelBadge.tsx`、`components/user/UserStats.tsx`。

**技术难点**：经验值事件的解耦注入（需要在 Community Module、Plugin Module 等多个模块的 Service 中注入经验值累积逻辑）。

**外部依赖**：无新增

---

## 2. 架构概览

```
个人主页 API 聚合:

GET /user/profile/:userId
       │
       ▼
  UserProfileService.getProfile(userId, currentUserId?)
       │
       ├── 1. prisma.user.findUnique({ id: userId })
       │
       ├── 2. 统计查询（并行）:
       │     ├── pluginCount = prisma.pluginProject.count({ where: { userId, isPublished: true } })
       │     ├── totalDownloads = prisma.pluginProject.aggregate({ _sum: { downloadCount } })
       │     ├── totalStars = prisma.pluginProject.aggregate({ _sum: { starCount } })
       │     └── followerCount = prisma.follow.count({ where: { followingId: userId } })
       │
       ├── 3. 作品列表:
       │     └── prisma.pluginProject.findMany({ where: { userId, isPublished: true }, orderBy: [{ isPinned: 'desc' }, { createdAt: 'desc' }] })
       │
       └── 4. isFollowing? (仅当 currentUserId 存在)
             └── prisma.follow.findUnique({ where: { followerId_followingId } })


经验值事件流:

UserLevelService.addExp(userId, amount, source)
       │
       ├── 1. prisma.user.update({ where: { id: userId }, data: { exp: { increment: amount } } })
       ├── 2. 检查是否升级
       ├── 3. 如升级 → WebSocket 推送 level.up 事件
       └── 4. 返回 { exp, level }
```

## 3. 数据库设计

> 本阶段使用已定义的 `User`、`PluginProject`、`Follow` 表，**无需新增 migration**。

**User 表新增字段**（需修改 Prisma Schema）：

```prisma
model User {
  // ... 已有字段 ...

  isPinnedPlugins Json?  // ← 新增：["uuid1", "uuid2"] 最多 3 个置顶插件 ID
}
```

> 实际上，置顶可以在 PluginProject 上加 `isPinned Boolean` 字段更直接。两者各有利弊：
>
> - User 存数组：一次查询即可获取所有置顶 ID，但更新需要全量写
> - PluginProject 加 `isPinned`：查询时按 `isPinned DESC` 排序，但需要在 count 时排除

**选择**：在 PluginProject 上加 `isPinned` 字段：

```prisma
// PluginProject 新增字段
model PluginProject {
  // ... 已有字段 ...
  isPinned    Boolean   @default(false)  // ← 新增
}
```

**需要新增 Migration**：

```bash
npx prisma migrate dev --name add_pinned_field
```

## 5. 核心逻辑

### 5.1 个人主页数据聚合 → AC-001 ~ AC-004

```typescript
// user-profile.service.ts
@Injectable()
export class UserProfileService {
  constructor(private prisma: PrismaService) {}

  async getProfile(userId: string, currentUserId?: string) {
    // 1. 获取用户基本信息
    const user = await this.prisma.user.findUnique({ where: { id: userId } });
    if (!user) throw new NotFoundException('User not found'); // → AC-101

    // 2. 并行统计查询
    const [pluginCount, downloadsAgg, starsAgg, followerCount, plugins, isFollowing] =
      await Promise.all([
        this.prisma.pluginProject.count({
          where: { userId, isPublished: true },
        }),
        this.prisma.pluginProject.aggregate({
          where: { userId, isPublished: true },
          _sum: { downloadCount: true },
        }),
        this.prisma.pluginProject.aggregate({
          where: { userId, isPublished: true },
          _sum: { starCount: true },
        }),
        this.prisma.follow.count({
          where: { followingId: userId },
        }),
        this.prisma.pluginProject.findMany({
          where: { userId, isPublished: true },
          orderBy: [{ isPinned: 'desc' }, { createdAt: 'desc' }],
          include: {
            ratings: { select: { score: true } },
          },
        }),
        currentUserId
          ? this.prisma.follow.findUnique({
              where: {
                followerId_followingId: {
                  followerId: currentUserId,
                  followingId: userId,
                },
              },
            })
          : Promise.resolve(null),
      ]);

    // 3. 计算均分
    const pluginsWithRating = plugins.map((p) => {
      const avgRating =
        p.ratings.length > 0 ? p.ratings.reduce((s, r) => s + r.score, 0) / p.ratings.length : 0;
      const { ratings, ...rest } = p;
      return { ...rest, avgRating };
    });

    // 4. 计算等级称号
    const title = this.getTitle(user.level);

    return {
      user: {
        id: user.id,
        nickname: user.nickname,
        avatar: user.avatar,
        bio: user.bio || '这个人很懒，什么都没有留下',
        level: user.level,
        exp: user.exp,
        title,
        expToNextLevel: user.level * 100, // BR-001
        createdAt: user.createdAt,
      },
      stats: {
        pluginCount,
        totalDownloads: downloadsAgg._sum.downloadCount || 0,
        totalStars: starsAgg._sum.starCount || 0,
        followerCount,
      },
      plugins: pluginsWithRating,
      isFollowing: !!isFollowing,
      isOwner: currentUserId === userId,
    };
  }

  private getTitle(level: number): string {
    if (level <= 5) return '新手创客';
    if (level <= 15) return '进阶工匠';
    if (level <= 30) return '资深开发者';
    return '大师';
  }
}
```

### 5.2 经验值引擎 → AC-006, AC-007, AC-009, AC-010, AC-201, AC-202

```typescript
// user-level.service.ts
const EXP_EVENTS = {
  plugin_generated: 10,
  plugin_published: 20,
  plugin_liked: 5,
  plugin_favorited: 3,
  user_signed_in: 5,
} as const;

type ExpEvent = keyof typeof EXP_EVENTS;

@Injectable()
export class UserLevelService {
  constructor(
    private prisma: PrismaService,
    private wsGateway: NotificationGateway,
  ) {}

  async addExp(
    userId: string,
    event: ExpEvent,
  ): Promise<{ exp: number; level: number; leveledUp: boolean }> {
    const amount = EXP_EVENTS[event];
    if (!amount) throw new BadRequestException(`Unknown exp event: ${event}`);

    const user = await this.prisma.user.findUnique({ where: { id: userId } });

    const newExp = user.exp + amount;
    const newLevel = this.calculateLevel(newExp);

    // 检测是否升级
    const leveledUp = newLevel > user.level;

    // 更新用户
    await this.prisma.user.update({
      where: { id: userId },
      data: { exp: newExp, level: newLevel },
    });

    // 升级时 WebSocket 推送
    if (leveledUp) {
      this.wsGateway.pushToUser(userId, 'level.up', {
        oldLevel: user.level,
        newLevel,
        title: this.getTitle(newLevel),
      });
    }

    return { exp: newExp, level: newLevel, leveledUp };
  }

  private calculateLevel(exp: number): number {
    // 等级公式：每级需 exp = level * 100，累计
    // Lv.1 → Lv.2: 100 exp
    // Lv.2 → Lv.3: 200 exp (累计 300)
    // Lv.3 → Lv.4: 300 exp (累计 600)
    // 解一元二次方程: sum(n) = n*(n+1)/2 * 100
    // 给定 exp，求最大 n 使得 n*(n+1)/2 * 100 <= exp
    let level = 1;
    for (let i = 1; i <= 50; i++) {
      const needed = i * 100;
      if (exp < needed) break;
      exp -= needed;
      level = i + 1;
    }
    return Math.min(level, 50);
  }

  private getTitle(level: number): string {
    if (level <= 5) return '新手创客';
    if (level <= 15) return '进阶工匠';
    if (level <= 30) return '资深开发者';
    return '大师';
  }
}
```

### 5.4 置顶作品管理 → AC-005, AC-103, AC-104

```typescript
// user-profile.service.ts — setPinnedPlugins()
async setPinnedPlugins(userId: string, pluginIds: string[]) {
  if (pluginIds.length > 3) {
    throw new BadRequestException('Maximum 3 pinned plugins');  // → AC-104
  }

  // 验证所有权 + 发布状态
  const plugins = await this.prisma.pluginProject.findMany({
    where: { id: { in: pluginIds }, userId },
  });
  if (plugins.length !== pluginIds.length) {
    throw new BadRequestException('Some plugins not found or not owned by you');
  }
  const unpublished = plugins.find(p => !p.isPublished);
  if (unpublished) {
    throw new BadRequestException(`Cannot pin unpublished plugin: ${unpublished.name}`);
  }

  // 批量更新置顶状态
  await this.prisma.$transaction(async (tx) => {
    // 清除所有置顶
    await tx.pluginProject.updateMany({
      where: { userId, isPinned: true },
      data: { isPinned: false },
    });
    // 设置新的置顶
    await Promise.all(
      pluginIds.map((id, index) =>
        tx.pluginProject.update({
          where: { id },
          data: { isPinned: true },
        }),
      ),
    );
  });
}
```

## 7. 技术决策

### 7.1 等级计算公式：累计求和而非固定阈值

**背景**：等级经验曲线的设计，影响用户升级的节奏感。

**选项**：

- A: 固定阈值（每级固定 500 exp）— 简单但后期升级太快
- B: 递增阈值（每级所需 exp = level × 100）— 前期升级快建立成就感，后期慢保持挑战性

**结论**：选择 B。`expForLevel(level) = level × 100`，累计求和。前期（1→2 级只需 100 exp = 生成 10 个插件）快速升级获得正反馈，后期（30→31 级需 3000 exp）需要持续投入。

### 7.2 置顶用 `isPinned` 字段而非 `order` 权重

**背景**：置顶作品需要排序，3 个置顶之间有顺序。

**结论**：选择简单方案——`isPinned: Boolean`，查询时 `orderBy: [{ isPinned: 'desc' }, { createdAt: 'desc' }]`。3 个置顶之间按发布时间排序，不引入额外的 `pinOrder` 字段。如果需要自定义顺序，后续再加。

### 7.3 统计数据实时查询而非缓存

**背景**：个人主页的统计数字（下载量、获赞数等）的获取方式。

**结论**：选择实时聚合查询。个人主页的访问频次不高（日均 PV 远低于作品广场），实时计算完全可接受。`_sum` 和 `_count` 在索引下毫秒级完成。

## 8. 安全与性能

**权限**：

- 个人主页所有数据公开可见（`@Public()`）
- 置顶管理仅自己（检查 `userId === currentUserId`）
- 经验值注入由各个 Service 内部调用，不暴露为 API

**性能**：

- `GET /user/profile/:id` 的 6 个并行 `Promise.all` 查询，总耗时 ≈ 最慢的那个
- PluginProject 的 `userId + isPublished` 复合索引保障查询性能
- 经验值写入每次只更新一个 User 表行，无性能瓶颈
