# 编译引擎 — 技术设计文档

## 1. 设计概要

**功能描述**：实现在 Docker 沙箱中对 AI 生成的 Minecraft 插件进行 Maven 编译，支持实时日志推送、异步队列处理、编译错误自动分析和 AI 修复。这是让 AI 生成的代码变为可运行 `.jar` 文件的最后关卡。

**影响范围**：后端新增 `compile/` 模块 + `docker/compiler/` 目录（Dockerfile）；前端新增 `components/plugin/CompileStatus.tsx`；修改 `pages/dashboard/PluginEdit.tsx` 增加编译按钮和日志面板。

**技术难点**：

- Docker 容器的并发管理和资源隔离（多用户同时编译互不干扰）
- Maven 编译日志的实时流式读取和推送
- 编译错误的 AI 修复——需要将 Maven 错误信息转换为精确的修复策略

**外部依赖**：`dockerode`（Node.js Docker SDK）、`@nestjs/bull`（Bull 队列）、`archiver`（已在 Phase 1.4 安装）

## 2. 架构概览

```
编译流程:

POST /compile/start/:pluginId
       │
       ▼
  CompileController
       │
       ├── 1. 校验配额 + 所有权
       ├── 2. 更新 PluginVersion.compileStatus = COMPILING
       ├── 3. Bull Queue: compileQueue.add({ pluginId, version })
       └── 4. 返回 { queued: true, position: N }
              │
              ▼ (异步消费)
  CompileProcessor (@Processor)
       │
       ├── 1. 读取插件配置（JDK 版本、源码路径、依赖 JAR 路径）
       ├── 2. DockerService.compile(pluginId, jdkVersion)
       │     ├── a. docker.createContainer({ Image: "plugin-compiler:jdk17", ... })
       │     ├── b. 复制源码到容器: tar stream
       │     ├── c. 挂载依赖 JAR 到 ~/.m2/repository
       │     ├── d. docker.container.start()
       │     ├── e. 流式读取 stdout/stderr → WebSocket push
       │     └── f. docker.container.wait() → exit code
       │
       ├── 3. exit code === 0 → SUCCESS
       │     ├── 复制 JAR 产物到 jars/ 目录
       │     └── 更新 compileStatus = SUCCESS
       │
       └── 4. exit code !== 0 → FAILED
             ├── 更新 compileStatus = FAILED
             ├── 保存完整 compileLog 到数据库
             └── WebSocket 推送失败事件


自动修复流程:

POST /compile/fix/:pluginId
       │
       ▼
  AutoFixService
       │
       ├── 1. 读取 compileLog + 当前代码
       ├── 2. 调用 DeepSeek API 分析错误并生成修复代码
       ├── 3. 应用修复 → 创建新版本（version+1）
       ├── 4. 检查修复轮次 ≤ 3
       └── 5. 自动触发重新编译
```

### 5.2 Dockerode 编译服务 → AC-001, AC-002, AC-003, AC-008, AC-202

```typescript
// docker.service.ts
@Injectable()
export class DockerService {
  private docker = new Docker();

  async compile(pluginId: string, userId: string): Promise<CompileResult> {
    const plugin = await this.prisma.pluginProject.findUnique({ where: { id: pluginId } });
    const imageTag = this.getImageTag(plugin.javaVersion); // "plugin-compiler:jdk17"

    const srcDir = this.getSrcDir(userId, pluginId);
    const depsDir = this.getDepsDir(userId, pluginId);

    // 创建 tar 包（包含源码 + 依赖 JAR）
    const tarStream = await this.createTarWithSource(srcDir, depsDir);

    const container = await this.docker.createContainer({
      Image: imageTag,
      Cmd: ['mvn', 'clean', 'package'],
      HostConfig: {
        AutoRemove: true, // → BR-002, AC-202
        Memory: 1024 * 1024 * 1024, // 1GB 内存限制
        CpuPeriod: 100000,
        CpuQuota: 50000, // 0.5 CPU
      },
      Env: ['MAVEN_OPTS=-Xmx512m'],
    });

    // 复制源码到容器
    await container.putArchive(tarStream, { path: '/app' });

    // 启动容器
    await container.start();

    // 流式读取日志 → WebSocket 推送
    const logStream = await container.logs({
      follow: true,
      stdout: true,
      stderr: true,
    });

    let fullLog = '';
    logStream.on('data', (chunk) => {
      const text = chunk.toString('utf8');
      fullLog += text;
      this.wsGateway.pushCompileProgress(pluginId, this.parseProgress(text), text);
    });

    // 等待退出码
    const { StatusCode } = await container.wait();
    // 容器已自动移除 (AutoRemove)

    // 如果编译成功，复制 JAR 产物
    if (StatusCode === 0) {
      const jarBuffer = await this.copyJarFromContainer(container, plugin.packageName);
      const jarPath = path.join(
        this.getJarsDir(userId, pluginId),
        `plugin-v${plugin.currentVersion}.jar`,
      );
      await fs.promises.writeFile(jarPath, jarBuffer);
    }

    return {
      success: StatusCode === 0,
      log: fullLog,
    };
  }

  private parseProgress(logLine: string): number {
    if (logLine.includes('Downloading')) return 20;
    if (logLine.includes('Compiling')) return 50;
    if (logLine.includes('Processing')) return 70; // annotation processing
    if (logLine.includes('jar')) return 90;
    if (logLine.includes('BUILD SUCCESS') || logLine.includes('BUILD FAILURE')) return 100;
    return 0; // 前端根据已知阶段推断
  }
}
```

### 5.3 Bull 队列处理器 → AC-001, AC-009, AC-101, AC-103

```typescript
// compile.processor.ts
@Processor('compile')
export class CompileProcessor {
  constructor(
    private dockerService: DockerService,
    private prisma: PrismaService,
    private wsGateway: ProgressGateway,
  ) {}

  @Process({ concurrency: 3 }) // 最多3个并发 → BR-006
  async handleCompile(job: Job<{ pluginId: string; userId: string }>) {
    const { pluginId, userId } = job.data;
    const timeout = 120_000; // 120s → AC-101

    try {
      // 更新状态为 COMPILING
      await this.updateStatus(pluginId, 'COMPILING');

      // 在 Promise.race 中执行编译
      const result = await Promise.race([
        this.dockerService.compile(pluginId, userId),
        this.delay(timeout).then(() => {
          throw new Error('Compilation timeout');
        }),
      ]);

      // 处理结果
      if (result.success) {
        await this.updateStatus(pluginId, 'SUCCESS', result.log);
        this.wsGateway.pushEvent(pluginId, 'compile.completed', { status: 'SUCCESS' });
      } else {
        await this.updateStatus(pluginId, 'FAILED', result.log);
        this.wsGateway.pushEvent(pluginId, 'compile.completed', { status: 'FAILED' });
      }
    } catch (error) {
      await this.updateStatus(pluginId, 'FAILED', error.message);
      this.wsGateway.pushEvent(pluginId, 'compile.completed', {
        status: 'FAILED',
        error: error.message,
      });
    }
  }
}
```

### 5.4 自动修复服务 → AC-005, AC-006, AC-102, AC-203

```typescript
// auto-fix.service.ts
@Injectable()
export class AutoFixService {
  private readonly MAX_FIX_ROUNDS = 3; // → BR-005, AC-203

  async fix(pluginId: string): Promise<FixResult> {
    const version = await this.prisma.pluginVersion.findFirst({
      where: { pluginId },
      orderBy: { version: 'desc' },
    });

    // 检查修复轮次
    const fixKey = `fix:${pluginId}:round`;
    const round = await this.redis.incr(fixKey);
    await this.redis.expire(fixKey, 3600); // 1h 过期

    if (round > this.MAX_FIX_ROUNDS) {
      throw new BadRequestException('Max fix rounds reached'); // → AC-102
    }

    // 调用 DeepSeek 修复
    const response = await this.openai.chat.completions.create({
      model: 'deepseek-v4-flash',
      messages: [
        {
          role: 'system',
          content: `你是 Minecraft 插件编译错误修复专家。
分析以下 Maven 编译错误，输出修复后的完整文件。
只修改有问题的部分，不修改无关代码。
输出格式要求：
// File: path/to/file.java
\`\`\`java
修复后的完整文件内容
\`\`\`

常见错误修复策略：
- "cannot find symbol" → 检查 import 语句，补充缺失的 import
- "incompatible types" → 检查类型转换
- "does not override" → 检查方法签名
- "cannot resolve constructor" → 检查构造参数
- plugin.yml 路径/类名错误 → 修正为实际类路径`,
        },
        {
          role: 'user',
          content: `编译错误日志：\n${version.compileLog}\n\n当前代码：\n${JSON.stringify(version.filesManifest)}`,
        },
      ],
      temperature: 0.2,
      max_tokens: 4096,
    });

    // 解析修复后的文件
    const fixedFiles = this.parseFixResponse(response.choices[0]?.message?.content || '');

    // 应用修复 → 创建新版本
    const newVersion = await this.versionService.createSnapshot(pluginId, {
      ...(version.filesManifest as Record<string, string>),
      ...fixedFiles, // 覆盖修复的文件
    });

    // 自动触发重新编译
    await this.compileQueue.add({ pluginId, userId: version.userId });

    return {
      newVersion: newVersion.version,
      fixRound: round,
      fixes: Object.keys(fixedFiles).map((f) => `Fixed: ${f}`),
      recompileStarted: true,
    };
  }
}
```

### 5.5 WebSocket 事件推送

**事件表**：

| 事件名                  | 方向          | Payload                              | 触发时机            |
| ----------------------- | ------------- | ------------------------------------ | ------------------- | -------- |
| `compile.queued`        | Server→Client | `{ pluginId, position: 0 }`          | 任务入队            |
| `compile.progress`      | Server→Client | `{ pluginId, percent, stage, log }`  | 编译中（逐行推送）  |
| `compile.completed`     | Server→Client | `{ pluginId, status: "SUCCESS"       | "FAILED", error? }` | 编译完成 |
| `compile.fix.completed` | Server→Client | `{ pluginId, newVersion, fixRound }` | 修复 + 重编译触发   |

### 5.6 前端 CompileStatus 组件 → AC-001 ~ AC-007

```tsx
// CompileStatus.tsx
function CompileStatus({ pluginId }: { pluginId: string }) {
  const [status, setStatus] = useState<CompileStatus>('IDLE');
  const [progress, setProgress] = useState(0);
  const [log, setLog] = useState('');
  const [fixRound, setFixRound] = useState(0);

  const ws = useWebSocket();

  useEffect(() => {
    ws.on(`compile.progress:${pluginId}`, (data) => {
      setProgress(data.percent);
      setLog((prev) => prev + data.log);
      setStatus('COMPILING');
    });

    ws.on(`compile.completed:${pluginId}`, (data) => {
      setStatus(data.status === 'SUCCESS' ? 'SUCCESS' : 'FAILED');
    });

    ws.on(`compile.fix:${pluginId}`, (data) => {
      setFixRound(data.fixRound);
    });
  }, []);

  const handleCompile = async () => {
    await apiClient.post(`/compile/start/${pluginId}`);
    setStatus('COMPILING');
  };

  const handleFix = async () => {
    await apiClient.post(`/compile/fix/${pluginId}`);
    setFixRound((r) => r + 1);
  };

  return (
    <div className="space-y-md">
      {/* 编译按钮 */}
      <Button onClick={handleCompile} disabled={status === 'COMPILING'}>
        {status === 'IDLE' && '编译'}
        {status === 'COMPILING' && `编译中… ${progress}%`}
        {status === 'SUCCESS' && '重新编译'}
        {status === 'FAILED' && '重新编译'}
      </Button>

      {/* 进度条 */}
      {status === 'COMPILING' && <ProgressBar percent={progress} />}

      {/* 日志面板 */}
      {(status === 'COMPILING' || status === 'SUCCESS' || status === 'FAILED') && (
        <LogViewer log={log} isError={status === 'FAILED'} />
      )}

      {/* 失败操作 */}
      {status === 'FAILED' && (
        <div className="flex gap-md">
          <Button variant="secondary" onClick={handleFix} disabled={fixRound >= 3}>
            {fixRound < 3 ? `自动修复 (第${fixRound + 1}轮)` : '已达最大修复次数'}
          </Button>
        </div>
      )}

      {/* 成功下载 */}
      {status === 'SUCCESS' && (
        <div className="flex gap-md">
          <Button onClick={() => download(`/file/download/${pluginId}/jar`)}>下载 JAR</Button>
          <Button variant="secondary" onClick={() => download(`/file/download/${pluginId}/zip`)}>
            下载源码 ZIP
          </Button>
        </div>
      )}
    </div>
  );
}
```

## 7. 技术决策

### 7.1 Docker 容器资源限制

**背景**：多用户并发编译时，防止单个容器耗尽服务器资源。

**选项**：

- A: 不限制资源 — 编译快，但一个用户的编译可能影响其他用户甚至拖垮服务器
- B: 限制每个容器 1GB 内存 + 0.5 CPU — 编译稍慢，但资源隔离安全

**结论**：选择 B。Dockerode 的 `HostConfig.Memory` 和 `CpuPeriod/CpuQuota` 限制每个容器资源。Minecraft 插件编译（小型 Maven 项目）在 1GB/0.5CPU 下通常 20-40s 完成，完全够用。

### 7.2 Bull 队列而非直接 Dockerode 调用

**背景**：编译是耗时操作（20-60s），需要异步处理。

**选项**：

- A: 直接在 HTTP 请求中等待 — 简单但阻塞连接，容易超时，不支持排队
- B: Bull 队列 — 异步执行，支持并发控制/重试/延迟/任务等待

**结论**：选择 B。Bull 队列天然支持 `concurrency: 3` 控制同时编译数量，任务排队、重试、超时处理都开箱即用。WebSocket 推送结果，HTTP 请求即时返回。

### 7.3 Maven 编译而非 Gradle

**背景**：构建工具选择。

**选项**：

- A: Gradle — 更现代，但 Minecraft 插件生态以 Maven 为主流
- B: Maven — 生态最成熟，Spigot/Paper 官方文档使用 Maven，AI Prompt 也以 Maven 为标准

**结论**：选择 B。所有 Prompt 模板生成的 `pom.xml` 都是 Maven 格式，与社区标准一致。

### 7.4 日志读取：`container.logs` 流式而非一次性读取

**背景**：编译日志的获取方式。

**选项**：

- A: 编译完成后一次性读取完整日志 — 简单但用户无法看到实时进度
- B: `container.logs({ follow: true, stdout: true, stderr: true })` 流式读取 — 实时推送

**结论**：选择 B。流式读取 + WebSocket 逐行推送，用户可见实时编译输出。日志内容超过 100KB 时前端只展示最近 50 行 + "显示全部"按钮。

## 8. 安全与性能

**容器安全**：

- 每个编译在独立容器中执行 → 隔离用户代码，防止恶意代码影响宿主机 → BR-002
- 容器资源限制 1GB RAM / 0.5 CPU → 防止 fork bomb 或内存溢出
- `AutoRemove: true` → 编译完成自动清理容器，不留僵尸容器 → AC-202
- 容器内无网络权限（可通过 `NetworkMode: 'none'` 配置），但 Maven 需要下载依赖，因此保留网络

**队列安全**：

- Bull 队列持久化在 Redis 中，服务器重启后任务不会丢失
- 编译超时 120s 自动标记失败 → AC-101
- 同一插件不能同时编译（检查 `compileStatus === COMPILING` 时阻止重复提交）

**自动修复安全**：

- 最多 3 轮修复 → 防止 AI 无限循环修复 → AC-102, AC-203
- 修复后创建新版本，不覆盖用户原有版本 → 用户可回退
