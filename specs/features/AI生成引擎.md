# AI 生成引擎 — 功能需求说明书

## 1. 背景与价值

**为什么做**：这是 PluginGen 的核心价值所在——用户不需要懂 Java，不需要配置 Maven，只要用自然语言描述"我想要一个签到插件"，AI 就能生成一个完整的、可编译的 Minecraft 插件项目。这是整个平台区别于普通代码编辑器的根本差异。

**做成什么样**：用户在工作台点击"创建新插件" → 填写表单（插件名、MC 版本、服务端核心、包名、功能描述） → 页面显示实时生成进度 → 生成完成后自动跳转到编辑页 → 用户查看代码 → 如果觉得不够好，可以输入"再加一个 reload 命令"进行增量修改 → 选中某段代码点击"AI 解释"查看说明。

**关联**：路线图 Phase 1.5 / 里程碑一（MVP）

## 2. 范围

### 本次做

- 后端：6 个 Prompt 模板（Bukkit/Spigot/Paper/Purpur/BungeeCord/Velocity），每个模板定义了生成该核心插件的完整骨架结构
- 后端：增量修改 Prompt 模板（接收当前代码 + 修改描述 → 输出修改后的完整文件）
- 后端：AI 代码解释 Prompt
- 后端：DeepSeek API 调用服务（OpenAI SDK 兼容模式）
- 后端：项目骨架构建器（解析 AI 返回的代码块，生成目录结构和物理文件）
- 后端：`POST /ai/generate` — 生成新插件（调用 AI → 解析响应 → 创建 PluginProject → 写入磁盘）
- 后端：`POST /ai/modify/:pluginId` — 增量修改（读取当前代码 + 修改描述 → 调用 AI → diff → 新版本）
- 后端：`POST /ai/explain/:pluginId/:filePath` — AI 解释指定文件的代码
- 后端：WebSocket 事件推送（生成进度：`ai.progress`、`ai.generated`、`ai.error`）
- 前端：AI 生成表单（CreatePlugin.tsx — 插件名、MC 版本、核心、Java 版本、包名、功能描述）
- 前端：实时进度条（"正在生成项目结构…" / "正在生成代码…" / "完成"）
- 前端：生成完成后自动跳转到编辑页
- 前端：增量修改入口（编辑页底部的修改描述输入框 + 提交按钮）
- 前端：AI 解释功能（选中代码或文件 → 点击"AI 解释" → 侧栏/弹出面板显示解释内容）

### 本次不做

- 多轮对话式生成（当前是一次生成，不支持后续追问）— 后续迭代
- AI 自动补全代码（Monaco 中智能提示）— 后续迭代
- 生成结果的用户手动编辑并保存 — Phase 1.4 的 Monaco 是只读的，编辑功能后续迭代
- 编译检测（生成后的自动编译）— Phase 1.6
- 自动修复编译错误 — Phase 1.6 的 auto-fix

## 4. 需求详述

### 4.1 核心流程

#### 流程 1：首次 AI 生成

1. 用户点击工作台的"创建新插件" → 进入 `/dashboard/create`
2. 用户填写表单：
   - **插件名称**（必填）：如 "DailyReward"
   - **Minecraft 版本**（必填）：下拉选择 1.8~最新版
   - **服务端核心**（必填）：下拉选择 Bukkit/Spigot/Paper/Purpur/BungeeCord/Velocity
   - **Java 版本**（根据核心+MC 版本自动匹配，也可手动修改）
   - **包名**（必填）：如 `com.example.dailyreward`
   - **功能描述**（必填，多行文本框）："一个每日签到插件，玩家每天可以签到一次，获得随机奖励。签到 7 天有额外奖励。需要一个 /reward 命令查看可领取的奖励。"
3. 用户点击"生成插件" → 按钮变为"生成中…"并置灰
4. 页面展示实时进度条（WebSocket 推送）：
   - "正在根据需求生成项目结构…"
   - "正在生成主类和命令注册…"
   - "正在生成事件监听器和配置…"
   - "正在生成项目配置文件…"
   - "生成完成！正在跳转到编辑页…"
5. 生成完成后自动跳转到 `/dashboard/plugins/:id/edit`
6. 用户可以在编辑页查看所有生成的文件

#### 流程 2：增量修改

1. 用户在编辑页底部找到修改输入框
2. 输入修改描述（如 "给签到指令增加一个 /reward top 排行榜子命令"）
3. 点击"应用修改" → 系统读取当前代码 + 修改描述 → 调用 AI → 生成修改后的代码
4. 修改完成后自动创建新版本，刷新文件树和编辑器内容
5. 如果修改失败（AI 返回格式不正确），显示错误提示并保留原代码

#### 流程 3：AI 代码解释

1. 用户在编辑页的文件树中选中一个文件（或选中编辑器中的一段代码）
2. 点击"AI 解释"按钮
3. 右侧或底部弹出解释面板
4. AI 逐段解释：这个文件的作用、主要类和方法的功能、命令和事件的注册方式、配置文件的读取逻辑等
5. 解释面板可以滚动和关闭

### 4.2 交互规则

- 表单中的 MC 版本选择后，Java 版本自动匹配：1.8→8、1.13→11、1.17→17、1.20→21，Velocity 固定 17
- 生成过程中按钮置灰不可重复点击，显示加载动画
- 生成总耗时预期 ≤ 30 秒（DeepSeek API 响应时间 + 解析时间）
- 增量修改完成后自动刷新编辑页的文件树和编辑器内容
- AI 解释面板支持 Markdown 渲染（代码行内高亮）
- 如果用户输入的描述过于模糊或缺少关键信息（如没说什么核心），在提交后 AI 会按默认值处理并在结果中标注

### 4.3 业务规则

| 编号   | 规则                                                                                                                                                                      |
| ------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| BR-001 | 不同服务端核心生成不同项目结构：Bukkit/Spigot/Paper/Purpur 使用 plugin.yml + JavaPlugin；BungeeCord 使用 plugin.yml + Plugin；Velocity 使用 @Plugin 注解（无 plugin.yml） |
| BR-002 | 生成的 pom.xml 必须包含对应核心的 Maven 依赖（spigot-api、paper-api 等），版本与用户选择的 MC 版本匹配                                                                    |
| BR-003 | 每次增量修改都创建新版本（`currentVersion + 1`），不覆盖已有版本                                                                                                          |
| BR-004 | AI 响应的代码块必须包含完整的文件路径注释（如 `// File: src/main/java/com/example/Main.java`），供骨架构建器解析                                                          |
| BR-005 | 生成失败时（AI 返回空或格式错误），不创建 PluginProject，返还一次配额给用户                                                                                               |

## 5. 验收标准

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001**: 填写生成表单并提交
  - **Given** 用户已登录且在 `/dashboard/create` 页面
  - **When** 填写所有必填字段（插件名 "DailyReward"、MC 版本 "1.20.1"、核心 "Paper"、包名 "com.example.dailyreward"、功能描述 "一个每日签到插件"）并点击"生成插件"
  - **Then** 按钮变为"生成中…"并置灰，进度条开始显示实时进度

- [ ] **AC-002**: 生成完成后跳转编辑页
  - **Given** AI 生成成功
  - **When** 生成进度达到 100%
  - **Then** 自动跳转到 `/dashboard/plugins/:id/edit`，页面显示新插件的文件树和默认打开的文件

- [ ] **AC-003**: 不同核心生成不同结构
  - **Given** 用户选择 Bukkit 核心生成一个插件，再选择 Velocity 核心生成另一个
  - **When** 查看两个项目的文件结构
  - **Then** Bukkit 项目包含 `plugin.yml` + `extends JavaPlugin` 的主类；Velocity 项目不包含 `plugin.yml`，主类使用 `@Plugin` 注解

- [ ] **AC-004**: 生成结果包含标准 Maven 结构
  - **Given** 生成完成
  - **When** 查看文件树
  - **Then** 包含 `pom.xml`（含对应核心依赖）、`src/main/java/.../` 下的主类和命令类、`plugin.yml`（如适用）、`config.yml`（如适用）

- [ ] **AC-005**: WebSocket 实时推送进度
  - **Given** 用户在生成过程中
  - **When** 查看页面
  - **Then** 进度条展示至少 3 个不同的阶段文案（如"正在生成项目结构"、"正在生成代码"、"正在保存项目"），进度百分比从 0 递增到 100

- [ ] **AC-006**: 增量修改追加功能
  - **Given** 用户已在编辑页查看生成的插件
  - **When** 在修改输入框中输入"增加一个 /reload 命令重新加载配置"并点击"应用修改"
  - **Then** 修改完成后，版本号 +1，文件树中 `Main.java` 新增了 `/reload` 命令的处理逻辑

- [ ] **AC-007**: 增量修改创建新版本
  - **Given** 当前版本为 v1
  - **When** 完成一次增量修改
  - **Then** 版本历史中出现 v2，v1 的内容可被回退查看

- [ ] **AC-008**: AI 解释代码
  - **Given** 用户在编辑页打开了一个文件
  - **When** 点击"AI 解释"按钮
  - **Then** 弹出解释面板，显示 AI 对该文件的逐段解释（如"Main.java 是插件主类，负责在 onEnable 中注册命令和事件；Command.java 处理 /reward 命令的逻辑……"），内容以 Markdown 格式渲染

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-101**: 未填写必填字段
  - **Given** 用户在生成表单
  - **When** 插件名或功能描述为空时点击"生成插件"
  - **Then** 按钮不可点击，对应输入框显示红色边框和"此项为必填"提示

- [ ] **AC-102**: 生成过程中 WebSocket 断开
  - **Given** AI 生成过程中 WebSocket 连接断开
  - **When** 网络恢复
  - **Then** 前端显示"连接已断开，请刷新页面查看生成结果"提示，系统后端继续完成生成

- [ ] **AC-103**: AI 返回格式错误
  - **Given** AI 返回的内容不包含标准文件路径注释
  - **When** 骨架构建器解析失败
  - **Then** 返回错误提示"AI 生成结果格式异常，请重试"，不创建项目，返还一次配额

- [ ] **AC-104**: AI 响应超时
  - **Given** DeepSeek API 超过 30 秒无响应
  - **When** 超时
  - **Then** 显示"AI 生成超时，请稍后重试"，返还一次配额

- [ ] **AC-105**: 增量修改描述无效
  - **Given** 用户在修改输入框中输入"123"（无意义的数字）
  - **When** 点击"应用修改"
  - **Then** 提示"请提供更详细的修改描述（至少 5 个字）"，不调用 AI

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-201**: 不同核心对应不同依赖版本 ← BR-002
  - **Given** 选择 Paper 核心 + MC 1.20.1
  - **When** 查看生成的 `pom.xml`
  - **Then** 包含 `paper-api` 依赖，版本为 `1.20.1-R0.1-SNAPSHOT` 或对应最新版本

- [ ] **AC-202**: 增量修改不覆盖旧版本 ← BR-003
  - **Given** 插件已有 v1
  - **When** 完成一次增量修改
  - **Then** 数据库 `PluginVersion` 表新增一条 `version=2` 的记录，`version=1` 的记录完整保留

- [ ] **AC-203**: 生成失败返还配额 ← BR-005
  - **Given** 用户当日剩余配额为 5
  - **When** AI 生成失败（返回格式错误）
  - **Then** 用户当日剩余配额仍为 5（未扣减）
