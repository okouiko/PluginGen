# 每日签到 & 任务系统 — 技术设计文档

## 1. 设计概要

**功能描述**：实现每日签到系统（Redis 防重复、连续天数计算、阶梯奖励）和每日任务引擎（4 种任务类型、事件钩子自动更新进度、任务奖励领取），配合全局配额显示组件提升用户留存和核心操作引导。

**影响范围**：后端新增 `daily/` 模块；前端新增 `pages/daily/DailyPage.tsx`、`components/user/QuotaDisplay.tsx`；修改 `components/layout/TopNav.tsx` 集成配额显示。

**技术难点**：

- 连续天数计算：需要判断昨天是否签到，跨 0 点正确处理
- 任务每日重置：基于日期字段区分今日/昨日任务，不依赖定时任务

**外部依赖**：无新增（Redis 已在 Phase 1.2 引入）

## 2. 架构概览

```
签到流程:

POST /daily/checkin
  → DailyController.checkin()
    → DailyService.checkin(userId)
      │
      ├── 1. Redis: SISMEMBER checkin:{date}:{userId}
      │     └── 已存在 → return { alreadyCheckedIn: true }  (→ AC-101)
      │
      ├── 2. Redis: SADD checkin:{date}:{userId}  (去重标记)
      │     Redis: EXPIRE checkin:{date}:{userId} 259200 (3 天自动过期)
      │
      ├── 3. 查询昨日签到 (prisma.dailyCheckIn)
      │     └── 存在 → streak = yesterday.streak + 1
      │     └── 不存在 → streak = 1
      │
      ├── 4. 创建签到记录: prisma.dailyCheckIn.create({ userId, date, streak })
      │
      ├── 5. 计算经验奖励:
      │     base = 5
      │     bonus7 = (streak % 7 === 0) ? 10 : 0
      │     bonus30 = (streak % 30 === 0) ? 30 : 0
      │     total = base + bonus7 + bonus30
      │
      ├── 6. UserLevelService.addExp(userId, total)
      │
      └── 7. 更新签到任务的进度（签到任务自动完成）
            → DailyTaskService.updateProgress(userId, 'checkin')


任务流程:

操作事件 → (如 AI 生成完成)
  → DailyTaskService.updateProgress(userId, 'generate_plugin')
    │
    ├── 1. 获取或创建今日任务记录
    ├── 2. progress += 1
    ├── 3. 如果 progress >= target → completed = true
    └── 4. 返回更新后的任务列表


每日重置:

不依赖 cron。每次查询时：
  GET /daily/tasks
    → 检查数据库中的任务记录日期
    → 如果是今天 → 返回已有记录
    → 如果是昨天（或更早）→ 创建今天的任务记录（progress = 0）
```

## 3. 数据库设计

> 本阶段使用 Phase 1.2 已定义的 `DailyCheckIn` 和 `DailyTask` 表，**无需新增 migration**。

**DailyCheckIn 表**：

| rewardClaimed | Boolean (default false) | 奖励是否已领取                                               |

**索引**：`@@unique([userId, date, taskType])` — 每人每天每种任务一条

## 5. 核心逻辑

### 5.1 签到核心 — Redis 防重复 + 连续天数 + 经验奖励 → AC-001 ~ AC-004, AC-101, AC-202

```typescript
// daily.service.ts
@Injectable()
export class DailyService {
  constructor(
    private prisma: PrismaService,
    private redis: RedisService,
    private userLevelService: UserLevelService,
    private taskService: DailyTaskService,
  ) {}

  async checkin(userId: string): Promise<CheckinResult> {
    const today = this.getTodayDate(); // 2026-06-18T00:00:00.000Z
    const dateKey = this.formatDateKey(today); // "2026-06-18"

    // 1. Redis 防重复检查 → AC-101, AC-201
    const redisKey = `checkin:${dateKey}:${userId}`;
    const alreadyChecked = await this.redis.sismember(redisKey, userId);
    if (alreadyChecked) {
      throw new ConflictException('Already checked in today');
    }

    // 2. Redis 记录签到（3 天 TTL）
    await this.redis.sadd(redisKey, userId);
    await this.redis.expire(redisKey, 259200); // 3 天

    // 3. 查询昨天签到 → 计算连续天数
    const yesterday = new Date(today);
    yesterday.setDate(yesterday.getDate() - 1);
    const yesterdayCheckin = await this.prisma.dailyCheckIn.findUnique({
      where: { userId_date: { userId, date: yesterday } },
    });
    const streak = yesterdayCheckin ? yesterdayCheckin.streak + 1 : 1; // → AC-004

    // 4. 创建签到记录
    await this.prisma.dailyCheckIn.create({
      data: { userId, date: today, streak },
    });

    // 5. 计算经验奖励
    const base = 5;
    const bonus7 = streak > 0 && streak % 7 === 0 ? 10 : 0; // → AC-002
    const bonus30 = streak > 0 && streak % 30 === 0 ? 30 : 0; // → AC-003
    const total = base + bonus7 + bonus30;

    const levelResult = await this.userLevelService.addExpRaw(userId, total);

    // 6. 签到任务自动完成 → AC-009
    await this.taskService.updateProgress(userId, 'checkin');
    await this.taskService.claimReward(userId, 'checkin');

    return {
      checkedIn: true,
      streak,
      expAwarded: total,
      bonusDetails: { base, streak7: bonus7, streak30: bonus30 },
      leveledUp: levelResult.leveledUp,
    };
  }

  private getTodayDate(): Date {
    const now = new Date();
    return new Date(Date.UTC(now.getFullYear(), now.getMonth(), now.getDate()));
  }

  private formatDateKey(date: Date): string {
    return date.toISOString().split('T')[0]; // "2026-06-18"
  }
}
```

### 5.2 每日任务引擎 → AC-005, AC-006, AC-104, AC-203, AC-204

```typescript
// daily-task.service.ts
const TASK_DEFINITIONS: TaskDefinition[] = [
  {
    taskType: 'generate_plugin',
    name: '生成 1 个插件',
    description: '使用 AI 生成一个 Minecraft 插件',
    target: 1,
    reward: 5,
  },
  {
    taskType: 'publish_plugin',
    name: '发布 1 个作品',
    description: '将你的插件发布到作品广场',
    target: 1,
    reward: 5,
  },
  {
    taskType: 'comment',
    name: '评论 1 次',
    description: '在作品广场发表一个评论',
    target: 1,
    reward: 5,
  },
  { taskType: 'checkin', name: '签到 1 次', description: '完成今日签到', target: 1, reward: 5 },
];

@Injectable()
export class DailyTaskService {
  constructor(
    private prisma: PrismaService,
    private userLevelService: UserLevelService,
  ) {}

  // 获取今日任务（如无则自动创建）→ AC-005, AC-104
  async getTodayTasks(userId: string) {
    const today = this.getTodayDate();
    const existingTasks = await this.prisma.dailyTask.findMany({
      where: { userId, date: today },
    });

    if (existingTasks.length > 0) {
      // 已有今日任务记录 → 直接返回
      return { date: today, tasks: this.enrichTasks(existingTasks) };
    }

    // 没有今日任务 → 创建（首次访问或跨天）→ AC-104
    const created = await this.prisma.$transaction(
      TASK_DEFINITIONS.map((def) =>
        this.prisma.dailyTask.create({
          data: { userId, date: today, taskType: def.taskType },
        }),
      ),
    );

    return { date: today, tasks: this.enrichTasks(created) };
  }

  // 更新任务进度 → AC-006, AC-203
  async updateProgress(userId: string, taskType: string) {
    const today = this.getTodayDate();
    const task = await this.prisma.dailyTask.findUnique({
      where: { userId_date_taskType: { userId, date: today, taskType } },
    });

    if (!task || task.completed) return; // 已完成不需要再更新

    const definition = TASK_DEFINITIONS.find((d) => d.taskType === taskType);
    if (!definition) return;

    const newProgress = task.progress + 1;
    const completed = newProgress >= definition.target;

    await this.prisma.dailyTask.update({
      where: { id: task.id },
      data: { progress: newProgress, completed },
    });
  }

  // 领取任务奖励 → AC-007
  async claimReward(userId: string, taskType: string) {
    const today = this.getTodayDate();
    const task = await this.prisma.dailyTask.findUnique({
      where: { userId_date_taskType: { userId, date: today, taskType } },
    });

    if (!task || !task.completed) throw new BadRequestException('Task not completed yet');
    if (task.rewardClaimed) throw new ConflictException('Reward already claimed');

    const definition = TASK_DEFINITIONS.find((d) => d.taskType === taskType);

    await this.prisma.dailyTask.update({
      where: { id: task.id },
      data: { rewardClaimed: true },
    });

    await this.userLevelService.addExpRaw(userId, definition.reward);

    return { taskType, reward: definition.reward, claimed: true };
  }

  private enrichTasks(tasks: DailyTask[]) {
    return tasks.map((task) => {
      const def = TASK_DEFINITIONS.find((d) => d.taskType === task.taskType);
      let status: 'in_progress' | 'ready' | 'done' = 'in_progress';
      if (task.completed && task.rewardClaimed) status = 'done';
      else if (task.completed) status = 'ready';

      return {
        ...task,
        name: def?.name || '',
        description: def?.description || '',
        target: def?.target || 1,
        reward: def?.reward || 0,
        status,
      };
    });
  }
}
```

## 7. 技术决策

### 7.1 签到防重复：Redis Set + 数据库 unique 约束双重保证

**背景**：防止用户同一天多次签到获取经验。

**结论**：Redis Set（`checkin:{date}:{userId}`）+ Prisma `@@unique([userId, date])` 双重防重复。Redis 用于 API 层快速拦截（毫秒级），数据库 unique 约束作为最终防线。Redis Key 设置 3 天 TTL 自动清理旧数据。

### 7.2 连续天数：数据库 streak 字段 + 昨日查询

**背景**：连续天数的计算方式。

**结论**：在 `DailyCheckIn` 表中存储每次签到时的 `streak` 值（而非实时计算）。签到时查询昨天的签到记录的 streak，+1 后写入今天的记录。这种方式一次查询即可获得准确值，且历史数据不会因补签等操作而改变。

### 7.3 任务引擎：事件钩子而非轮询

**背景**：任务进度更新的触发方式。

**结论**：在相关 Service 中直接调用 `dailyTaskService.updateProgress()`。相比定时轮询数据库，事件钩子方式实时性更好（操作完成即刻更新），代码也更简洁。

## 8. 安全与性能

**Redis 防重复**：签到使用 Redis Set + 3 天 TTL，减少数据库写入压力。即使 Redis 宕机，数据库 unique 约束也能防止重复签到。

**任务创建时机**：任务记录在用户首次查看 `/daily` 时创建（懒加载），而非每天 0 点批量创建，避免 cron 的复杂性。

**无定时任务**：不依赖 cron job。所有"每日重置"逻辑通过查询时的日期判断实现。
