# 前端交互打磨 — 技术设计文档

## 1. 设计概要

**功能描述**：对 PluginGen 前端进行体验升级——响应式布局适配（3 断点）、暗色/亮色模式切换（DESIGN.md 完整 token 映射）、Framer Motion 页面过渡动画、移动端汉堡菜单、无障碍支持（aria-label/键盘导航/focus 样式/对比度）。

**影响范围**：仅前端。新增 `lib/theme.ts`（暗色主题管理器）、`components/layout/MobileNav.tsx`（汉堡菜单）；修改 `App.tsx`（AnimatePresence 路由动画）、`tailwind.config.ts`（暗色模式 token）、`components/layout/TopNav.tsx`（主题切换+汉堡图标）。

**技术难点**：暗色模式 token 的完整双向映射（每个亮色 token 都需要对应的暗色值）。

**外部依赖**：`framer-motion`（需新增安装）

**ThemeManager 实现**：

```typescript
// lib/theme.ts
type Theme = 'light' | 'dark' | 'system';

export function useTheme() {
  const [theme, setThemeState] = useState<Theme>(() => {
    return (localStorage.getItem('theme') as Theme) || 'system';
  });

  // 应用主题
  useEffect(() => {
    const root = document.documentElement;

    const applyTheme = (isDark: boolean) => {
      root.classList.toggle('dark', isDark);
    };

    if (theme === 'system') {
      const mq = window.matchMedia('(prefers-color-scheme: dark)');
      applyTheme(mq.matches);

      const handler = (e: MediaQueryListEvent) => applyTheme(e.matches);
      mq.addEventListener('change', handler); // → AC-101
      return () => mq.removeEventListener('change', handler);
    } else {
      applyTheme(theme === 'dark');
    }
  }, [theme]);

  const setTheme = (newTheme: Theme) => {
    setThemeState(newTheme);
    localStorage.setItem('theme', newTheme); // → AC-005
  };

  const toggleTheme = () => {
    const isDark = document.documentElement.classList.contains('dark');
    setTheme(isDark ? 'light' : 'dark');
  };

  return { theme, setTheme, toggleTheme, isDark: theme === 'dark' };
}
```

**组件中使用**：


### 5.3 页面过渡动画 → AC-006, AC-102

```tsx
// components/shared/PageTransition.tsx
'use client';
import { motion } from 'framer-motion';

export function PageTransition({ children }: { children: React.ReactNode }) {
  const prefersReducedMotion = window.matchMedia('(prefers-reduced-motion: reduce)').matches;

  if (prefersReducedMotion) return <>{children}</>; // → AC-102

  return (
    <motion.div
      initial={{ opacity: 0, y: 8 }}
      animate={{ opacity: 1, y: 0 }}
      exit={{ opacity: 0, y: -8 }}
      transition={{ duration: 0.2, ease: 'easeInOut' }}
    >
      {children}
    </motion.div>
  );
}

// 在 App.tsx 中使用
<AnimatePresence mode="wait">
  <Routes location={location} key={location.pathname}>
    <Route
      path="/"
      element={
        <PageTransition>
          <Home />
        </PageTransition>
      }
    />
    <Route
      path="/workspace"
      element={
        <PageTransition>
          <WorkspacePage />
        </PageTransition>
      }
    />
    {/* ...其余路由 */}
  </Routes>
</AnimatePresence>;
```

## 7. 技术决策

### 7.1 darkMode: 'class' 策略

**背景**：TailwindCSS 暗色模式的实现方式。

**选项**：

- A: `media` 策略 — 自动跟随系统，无法用户手动切换
- B: `class` 策略 — 通过 `<html class="dark">` 控制，支持系统跟随 + 手动切换

**结论**：选择 B。`class` 策略支持三状态（亮色/暗色/跟随系统），灵活性最高。

### 7.2 Framer Motion 而非 CSS 动画

**背景**：页面过渡动画的实现方式。

**选项**：

- A: CSS `@keyframes` + `transition` — 零依赖，但无法实现进入/离开动画的协调（AnimatePresence）
- B: Framer Motion — 需要引入新依赖（~30KB gzip），但 AnimatePresence 的 exit 动画、stagger、layout 动画开箱即用

**结论**：选择 B。`AnimatePresence` 的 exit 动画是实现页面切换淡出效果的关键能力，CSS 无法做到同等效果。30KB gzip 对于用户体验的提升是值得的。

### 7.3 暗色 token 不全部重写 CSS 变量，而是用 Tailwind dark: 前缀

**背景**：暗色模式下的颜色管理方式。

**选项**：

- A: CSS 变量（`--bg-canvas`）+ `var()` 引用 — 灵活但运行时开销稍大
- B: Tailwind `dark:` 前缀 — 编译时静态生成，零运行时开销

**结论**：选择 B。`dark:bg-surface-dark dark:text-on-dark` 在每个组件中显式写，虽然不如 CSS 变量"一处定义全局生效"优雅，但编译后零运行时开销、类型安全、IDE 自动补全。所有页面组件逐一添加 `dark:` 类。

## 8. 安全与性能

**动画性能**：

- `opacity` 和 `transform`（`translateY` / `scale`）是 GPU 加速属性，不影响 Layout/ Paint
- Framer Motion 的 `layout` 动画使用 `transform` 而非 `width/height`
- `AnimatePresence mode="wait"` 确保前一个页面卸载后才挂载下一个，避免内存泄漏

**暗色模式切换性能**：

- Tailwind `class` 策略仅切换 `<html>` 的一个 class，触发全量 CSS 重计算
- 首次切换时浏览器有一次样式重新计算（约 5-15ms），在 60fps 范围内
- 后续切换由于样式已缓存，耗时更短

**无障碍**：

- 所有动画尊重 `prefers-reduced-motion` → AC-102
- focus-visible 确保仅键盘用户看到 focus 环，鼠标点击时不显示 → AC-007
- 颜色对比度符合 WCAG AA（4.5:1）→ AC-104
