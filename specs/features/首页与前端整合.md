# 首页 & 前端核心页面整合 — 功能需求说明书

## 1. 背景与价值

**为什么做**：前 6 个 Phase 搭建了完整的功能链路（脚手架→数据库→认证→插件管理→AI 生成→编译），但用户访问网站首先看到的是首页——如果首页不好看、不清楚平台做什么，用户可能不会注册。同时，工作台入口、全局 Toast/ErrorBoundary/Loading 等体验设施也需要整合，让整个 MVP 形成一个可演示、可交付的完整产品。

**做成什么样**：用户打开 PluginGen.com → 看到暖色调奶油风格的首页（Hero 标题 + 功能介绍 + CTA 按钮）→ 点击"开始使用" → 进入工作台 → 注册/登录后的全链路体验顺畅无阻（全局 Toast 通知、Loading 状态、错误边界保护）。

**关联**：路线图 Phase 1.7 / 里程碑一（MVP）— 🚩 MVP 完成标志

## 2. 范围

### 本次做

- 前端：首页 `pages/Home.tsx`（严格按 DESIGN.md 规范）
  - Hero 区：大标题 + 副标题 + CTA 按钮（"开始创建插件" / "了解更多"）
  - 功能介绍区：3 个 feature-card 展示核心能力（AI 生成 / 在线编译 / 社区分享）
  - 代码窗口卡片：展示一段插件代码的暗色 mockup
  - CTA 区：珊瑚色全幅 CTA 条幅（"准备好创建你的第一个插件了吗？"）
  - 底部 Footer：复用已实现的 Footer 组件
- 前端：工作台首页 `/dashboard` 入口
  - 概览：用户昵称 + 等级 + 每日剩余配额
  - 快捷操作："创建新插件"大按钮
  - 最近项目列表（最近 5 个）
- 前端：整合编辑页三栏布局（文件树 | 编辑器 | 版本侧栏 + 编译状态）
- 前端：全局 Toast 通知系统（集成 `Sonner` 或 `react-hot-toast`）
- 前端：ErrorBoundary 全局错误捕获（显示"出错了"占位 + 刷新按钮）
- 前端：全局 Loading 状态（路由切换时的骨架屏 / spinner）
- 前端：404 页面（"页面不存在" + 返回首页按钮）
- 前端：响应式适配（768px 断点 — 导航汉堡菜单 / 三栏变单栏）

### 本次不做

- 动画效果（Framer Motion 入场动画）— Phase 3.1
- 暗色/亮色模式切换 — Phase 3.1
- 多语言切换 — 已确认不做
- 用户偏好设置页的深度定制 — 后续迭代
- 性能优化（代码分割、懒加载）— Phase 3.2
- 页面访问统计 — 后续迭代

## 4. 需求详述

### 4.1 核心流程

#### 流程 1：首页访问

1. 访客输入 `plugingen.com` → 进入首页 `/`
2. 页面渲染 Hero 区：Display XL 标题（如"用 AI 创建你的 Minecraft 插件"）+ 副标题 + 两个 CTA 按钮
3. 向下滚动 → 三个 Feature Card 介绍 AI 生成、在线编译、版本管理
4. 再向下 → 深色代码窗口卡片展示一段示例插件代码
5. 再向下 → 珊瑚色 CTA 区 + 按钮
6. 底部 → 深色 Footer

#### 流程 2：用户从首页到使用

1. 访客点击首页 CTA "开始创建" → 如果未登录，跳转 `/register`；如果已登录，跳转 `/dashboard`
2. 注册/登录后进入工作台 → 显示概览信息 + 创建按钮 + 最近项目

#### 流程 3：全局操作反馈

1. 用户执行任意操作（生成、编译、删除等）→ 成功/失败/信息性 Toast 提示
2. 错误被 ErrorBoundary 捕获 → 显示错误页面（非白屏）
3. 路由切换 / 数据加载中 → 显示骨架屏或 spinner

### 4.2 交互规则

- **首页**严格遵循 DESIGN.md 的排版和间距体系（96px section 间距、32px 卡片 padding）
- **首页 Hero 区标题**使用 Copernicus 衬线字体（display-xl/xl 级别），不粗体
- **首页配色交替节奏**：cream canvas → cream feature card → dark code window → coral CTA → dark footer
- **CTA 按钮**：珊瑚色（`bg-primary`），悬停变深（`bg-primary-active`）
- **工作台概览**：简化为卡片式布局，不是仪表盘
- **Toast**：右上角弹出，3 秒自动消失，支持成功/错误/信息三种类型
- **ErrorBoundary**：捕获渲染错误后，显示"出错了" + 错误描述 + "刷新页面"按钮
- **Loading**：列表加载使用骨架屏（灰色脉冲块），页面切换使用顶部进度条（NProgress 风格）或中心 spinner

### 4.3 业务规则

| 编号   | 规则                                                                                |
| ------ | ----------------------------------------------------------------------------------- |
| BR-001 | 首页不需要登录即可访问                                                              |
| BR-002 | 工作台 `/dashboard` 需要登录，未登录跳转 `/login`                                   |
| BR-003 | 404 页面无论登录状态均显示，不跳转                                                  |
| BR-004 | Toast 同一时间最多显示 3 条，超出时移除最旧的                                       |
| BR-005 | ErrorBoundary 仅捕获渲染阶段的错误，不捕获异步错误（异步错误由各自 try/catch 处理） |

## 5. 验收标准

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001**: 首页 Hero 区渲染
  - **Given** 访客访问首页
  - **When** 页面加载完成
  - **Then** 顶部 Hero 区显示 Display XL 标题（Copernicus 衬线字体）、副标题（StyreneB 无衬线字体）、两个 CTA 按钮（"开始创建"珊瑚色 + "了解更多"次要按钮）

- [ ] **AC-002**: 首页 Feature Cards
  - **Given** 访客向下滚动
  - **When** 看到中间功能区域
  - **Then** 三个 Feature Card 并排展示，背景色为 `bg-surface-card`（#efe9de），每个卡片包含一个图标、标题和描述文字

- [ ] **AC-003**: 首页深色代码窗口卡片
  - **Given** 访客继续向下滚动
  - **When** 看到代码展示区
  - **Then** 一个深色代码窗口卡片（`bg-surface-dark` #181715）展示一段示例插件代码（JetBrains Mono 字体，语法高亮）

- [ ] **AC-004**: 首页 CTA 跳转
  - **Given** 访客在首页点击"开始创建"按钮
  - **When** 未登录状态
  - **Then** 跳转到 `/register` 页面
  - **When** 已登录状态
  - **Then** 跳转到 `/dashboard` 页面

- [ ] **AC-005**: 工作台概览
  - **Given** 用户已登录且访问 `/dashboard`
  - **When** 页面加载完成
  - **Then** 显示：用户昵称和头像、等级和经验值、每日剩余配额（"剩余 18/20"）、一个大的"创建新插件"按钮、最近 5 个插件列表（如果有）

- [ ] **AC-006**: 工作台最近项目列表
  - **Given** 用户已有插件项目
  - **When** 查看工作台
  - **Then** 最近项目列表显示每个插件的名称、核心类型标签、最后更新时间

- [ ] **AC-007**: 全局 Toast 通知
  - **Given** 用户执行某个操作（如删除插件）
  - **When** 操作成功
  - **Then** 右上角弹出绿色 Toast："插件已删除"，3 秒后自动消失
  - **When** 操作失败
  - **Then** 右上角弹出红色 Toast："操作失败，请重试"

- [ ] **AC-008**: ErrorBoundary 显示
  - **Given** 某个页面组件抛出渲染错误
  - **When** ErrorBoundary 捕获
  - **Then** 显示友好错误页面："出错了" + 错误类型说明 + "刷新页面"按钮，页面非白屏、非空白

- [ ] **AC-009**: Loading 状态
  - **Given** 用户在切换路由或加载数据
  - **When** 数据尚未就绪
  - **Then** 页面显示 spinner 或骨架屏（灰色脉冲矩形块），不显示空白的闪烁内容

- [ ] **AC-010**: 404 页面
  - **Given** 用户访问一个不存在的路径（如 `/nonexistent`）
  - **When** 页面渲染
  - **Then** 显示"页面不存在" + 说明文字 + "返回首页"按钮

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-101**: 移动端响应式布局
  - **Given** 浏览器宽度 < 768px
  - **When** 访问首页
  - **Then** 导航栏变为汉堡菜单，Hero 区标题字号减小（64px → 32px），三个 Feature Card 堆叠为单列，代码窗口可横向滚动

- [ ] **AC-102**: 工作台无项目空状态
  - **Given** 新注册用户，还没有任何插件
  - **When** 访问工作台
  - **Then** 最近项目区域显示"还没有插件"空态提示 + "创建第一个插件"按钮

- [ ] **AC-103**: Toast 过多时的行为
  - **Given** 同一时间触发了多条 Toast
  - **When** Toast 数量超过 3
  - **Then** 只显示最近 3 条，旧的自动关闭

- [ ] **AC-104**: 网络错误时的 ErrorBoundary
  - **Given** 页面渲染时发生网络请求错误被 ErrorBoundary 捕获
  - **When** 错误页面展示
  - **Then** "刷新页面"按钮可用，点击后重新加载当前页面

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-201**: 首页无需登录 ← BR-001
  - **Given** 访客未登录
  - **When** 访问 `/`
  - **Then** 首页正常渲染，不跳转到登录页

- [ ] **AC-202**: 工作台需登录 ← BR-002
  - **Given** 访客未登录
  - **When** 访问 `/dashboard`
  - **Then** 自动跳转到 `/login?redirect=/dashboard`

- [ ] **AC-203**: 404 不跳转 ← BR-003
  - **Given** 访客或已登录用户
  - **When** 访问不存在的路径
  - **Then** 显示 404 页面，不跳转
