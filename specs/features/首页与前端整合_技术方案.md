# 首页 & 前端核心页面整合 — 技术设计文档

## 1. 设计概要

**功能描述**：实现 PluginGen 的首页（按 DESIGN.md 规范）、工作台概览页、全局 UX 设施（Toast、ErrorBoundary、Loading、404 页面）和响应式适配。这是 MVP 的最后一块拼图，让前 6 个 Phase 的功能整合成一个完整、可演示的产品。

**影响范围**：仅前端。新增 `pages/Home.tsx`、`pages/NotFound.tsx`，修改 `App.tsx`（路由、ErrorBoundary、Toast）、`pages/dashboard/DashboardLayout.tsx`（工作台概览），创建全局 `components/shared/ErrorBoundary.tsx` 和 `LoadingSpinner.tsx`。

**技术难点**：无（全是标准 React + TailwindCSS 页面开发，DESIGN.md 规范已有完整 token 映射）。

**外部依赖**：`react-hot-toast`（或 `sonner`，已在 Phase 1.1 安装）、`lucide-react`（图标，已在 Phase 1.1 安装）

### 5.3 全局 ErrorBoundary → AC-008, AC-104

```tsx
// components/shared/ErrorBoundary.tsx
interface ErrorBoundaryState {
  hasError: boolean;
  error: Error | null;
}

export class ErrorBoundary extends React.Component<
  { children: React.ReactNode },
  ErrorBoundaryState
> {
  state: ErrorBoundaryState = { hasError: false, error: null };

  static getDerivedStateFromError(error: Error): ErrorBoundaryState {
    return { hasError: true, error };
  }

  render() {
    if (this.state.hasError) {
      return (
        <div className="min-h-screen flex items-center justify-center bg-canvas">
          <div className="text-center p-xl">
            <AlertTriangle className="w-16 h-16 text-warning mx-auto mb-lg" />
            <h1 className="font-display text-display-sm text-ink mb-md">出错了</h1>
            <p className="text-body-md text-body mb-lg">页面遇到了意外错误，请刷新后重试。</p>
            <p className="text-body-sm text-muted mb-lg font-mono">{this.state.error?.message}</p>
            <Button onClick={() => window.location.reload()}>刷新页面</Button>
          </div>
        </div>
      );
    }
    return this.props.children;
  }
}
```

## 7. 技术决策

### 7.1 ErrorBoundary 使用 Class Component 而非 Function Component

**背景**：React 的 ErrorBoundary 目前只能用 Class Component 实现（Function Component 尚无 `componentDidCatch` 等效 API）。

**选项**：

- A: 使用 react-error-boundary 第三方库 — 提供 Hook 友好的 ErrorBoundary 封装
- B: 手写 Class Component ErrorBoundary — 零依赖，~30 行代码

**结论**：选择 B。手写 Class Component 实现 ErrorBoundary 非常标准且稳定，无需引入额外依赖。

### 7.2 Toast 使用 react-hot-toast 而非 sonner

**背景**：Shadcn/ui 官方推荐 sonner，但 react-hot-toast 更成熟。

**结论**：使用 react-hot-toast（已在 Phase 1.1 安装）。两者功能几乎相同，本项目已安装了 react-hot-toast，保持一致即可。

### 7.3 骨架屏使用纯 CSS 而非三方库

**背景**：Loading 骨架屏的实现方案。

**选项**：

- A: 使用 `react-loading-skeleton` — 开箱即用，但多一个依赖
- B: 手写 CSS `animate-pulse` + Tailwind — Tailwind 内置 `animate-pulse` 类，无需额外依赖

**结论**：选择 B。TailwindCSS v4 已内置 `animate-pulse`，加上自定义 `Skeleton` 组件完全够用。

## 8. 安全与性能

**性能**：

- 首页使用静态页面，无 API 调用，首次加载即渲染完成
- Feature Card 图标使用 `lucide-react` tree-shaking（只引入用到的图标）
- 代码窗口仅显示静态示例代码，无运行时高亮（不引入 Shiki 等重型高亮库）

**安全**：

- 首页和 404 页面无需认证，不暴露任何用户数据
- 工作台概览依赖已有的 `RequireAuth` 守卫
