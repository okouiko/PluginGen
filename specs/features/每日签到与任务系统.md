# 每日签到 & 任务系统 — 功能需求说明书

## 1. 背景与价值

**为什么做**：签到和每日任务是用户留存的经典手段——签到让用户每天都想回来，连续签到奖励制造"不能断"的心理驱动力；每日任务引导用户完成核心操作（生成插件、发布作品、评论等），既帮助用户熟悉功能，又增加了平台的核心指标。配合创作者等级体系，签到和任务获得的经验值让等级提升更快，形成"每天回来→完成任务→升级→成就感"的正循环。

**做成什么样**：用户进入签到页面 → 看到日历式签到记录 → 点击"签到"获得经验 +5 + 连续天数展示 → 下方显示今日任务列表（"生成 1 个插件"、"发布 1 个作品"、"评论 1 次"）→ 每个任务有进度条 → 完成后可领取额外经验奖励 → 导航栏显示每日剩余次数。

## 2. 范围

### 本次做

- 后端：签到 API（`POST /daily/checkin` — 签到 + 连续天数计算 + 经验奖励 + Redis 防重复）
- 后端：连续签到奖励规则（连续 7 天额外 +10，连续 30 天额外 +30）
- 后端：每日任务引擎
  - 4 种任务类型：`generate_plugin`（生成 1 个插件）、`publish_plugin`（发布 1 个作品）、`comment`（评论 1 次）、`checkin`（签到 1 次）
  - 任务进度跟踪：通过事件钩子在各个 Service 中更新进度
  - 任务每日重置
- 后端：`GET /daily/tasks` — 今日任务列表 + 进度
- 后端：`POST /daily/tasks/:id/claim` — 领取任务奖励（经验值）
- 后端：每日配额 Redis 中间件（已有，整合到签到页面展示）
- 前端：`DailyPage.tsx`（签到卡片 + 连续天数 + 任务列表 + 进度条 + 领取按钮）
- 前端：导航栏每日配额显示组件（`QuotaDisplay.tsx` — "剩余 15/20"）

### 本次不做

- 签到补签功能（错过一天后可用道具补签）— 后续迭代
- 不同类型的连续签到奖励进阶（如 7 天/14 天/30 天不同奖励档次）— 已有基础版
- 排行榜（连续签到天数排行）— 后续迭代
- 任务刷新重置的倒计时显示 — 后续迭代
- 通知提醒签到 — 后续迭代

## 4. 需求详述

### 4.1 核心流程

#### 流程 1：签到

1. 用户进入 `/daily` 签到页面
2. 页面顶部展示签到卡片：
   - 今日日期
   - "签到"大按钮
   - 如果已签到，"已签到"灰色不可点击
   - 连续签到天数（"已连续签到 5 天"）
3. 用户点击"签到"按钮
4. 后端检查：今天是否已签到（Redis Set 防重复）
5. 未签到 → 创建签到记录 → 计算连续天数 → 奖励经验 +5 → 连续 7 天额外 +10 → 连续 30 天额外 +30
6. 前端更新：按钮变为"已签到"，连续天数 +1，经验值增加，动画庆祝

#### 流程 2：查看与完成任务

1. 签到卡片下方展示今日任务列表（4 个）
2. 每个任务显示：任务名称、描述、当前进度/目标（如 "1/1"）、进度条、经验奖励值、状态（进行中/可领取/已完成）
3. 用户执行对应操作（生成插件、发布作品等）时，进度自动增加
4. 进度达到目标后，任务状态变为"可领取"
5. 用户点击"领取"按钮 → 获得奖励经验值 → 状态变为"已完成"
6. 所有任务完成后，显示"今日任务全部完成！"庆祝状态

### 4.2 交互规则

- 签到按钮：未签到→珊瑚色大按钮"签到"，已签到→灰色"已签到"不可点击
- 签到后显示简单的动效（如对勾动画或粒子效果）
- 任务进度条：0% 灰色 → 100% 绿色（完成后变为可领取状态）
- 领取按钮：红色角标提示"可领取"
- 连续天数展示：数字大号显示，附带火焰图标，醒目
- 导航栏配额显示：文字格式"剩余 15/20"，低于 5 次时变为红色
- 每日 0 点自动重置签到和任务（基于 Redis TTL + 数据库日期字段）

### 4.3 业务规则

| 编号   | 规则                                                                                 |
| ------ | ------------------------------------------------------------------------------------ |
| BR-001 | 每人每天只能签到一次，Redis 防重复（Key: `checkin:{userId}:{date}`）                 |
| BR-002 | 签到奖励：基础 +5 exp；连续 7 天额外 +10 exp；连续 30 天额外 +30 exp（奖励可叠加）   |
| BR-003 | 连续天数判断：如果昨天也签到了 → streak+1；否则 streak=1                             |
| BR-004 | 每日任务 4 种：generate_plugin(1/1)、publish_plugin(1/1)、comment(1/1)、checkin(1/1) |
| BR-005 | 任务进度在用户操作时自动更新（通过 Service 事件钩子），无需手动刷新页面              |
| BR-006 | 每个任务的奖励：+5 exp，签到任务签到后自动完成并奖励（无需手动领取）                 |

## 5. 验收标准

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001**: 签到成功
  - **Given** 用户已登录且在 `/daily` 页面
  - **When** 点击"签到"按钮
  - **Then** 按钮变为"已签到"（灰色不可点击），经验值 +5，连续天数 +1

- [ ] **AC-002**: 连续 7 天签到额外奖励
  - **Given** 用户已连续签到 6 天
  - **When** 第 7 天签到
  - **Then** 经验值 +5（基础）+ 10（连续 7 天奖励）= 总计 +15，显示"🎉 连续 7 天签到！额外 +10 exp"

- [ ] **AC-003**: 连续 30 天签到额外奖励
  - **Given** 用户已连续签到 29 天
  - **When** 第 30 天签到
  - **Then** 经验值 +5（基础）+ 30（连续 30 天奖励）= 总计 +35

- [ ] **AC-004**: 连续天数归零
  - **Given** 用户昨天签到但前天没签到
  - **When** 今天签到
  - **Then** 连续天数显示为 1（中断后重置）

- [ ] **AC-005**: 任务列表展示
  - **Given** 用户访问 `/daily` 页面
  - **When** 页面加载
  - **Then** 展示 4 个任务卡片，每个包含任务名称、描述（如"生成 1 个插件"）、进度（0/1）、奖励（+5 exp）、状态（进行中）

- [ ] **AC-006**: 任务进度自动更新
  - **Given** 用户的任务列表中"生成 1 个插件"进度为 0/1
  - **When** 用户完成一次 AI 生成
  - **Then** 任务进度变为 1/1，状态变更为"可领取"

- [ ] **AC-007**: 领取任务奖励
  - **Given** 任务状态为"可领取"
  - **When** 用户点击"领取"按钮
  - **Then** 按钮变为"已完成"，经验值 +5，显示"奖励已领取"提示

- [ ] **AC-008**: 导航栏配额显示
  - **Given** 用户已登录
  - **When** 查看导航栏
  - **Then** 显示"剩余 X/20"，X 为当日剩余生成次数

- [ ] **AC-009**: 签到任务自动完成
  - **Given** 用户今日的签到任务是"签到 1 次"
  - **When** 用户完成签到
  - **Then** 签到任务自动变为"已完成"，经验值 +5（无需手动领取）

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-101**: 重复签到
  - **Given** 用户今天已经签到过了
  - **When** 再次点击"签到"按钮
  - **Then** 按钮已置灰不可点击，如果通过 API 访问则返回 409 "Already checked in today"

- [ ] **AC-102**: 连续天数跨天计算
  - **Given** 用户今天签到（连续 N 天）
  - **When** 明天 00:05 用户签到
  - **Then** 连续天数变为 N+1（跨天计算正确）

- [ ] **AC-103**: 任务全部完成
  - **Given** 用户完成了全部 4 个任务
  - **When** 查看任务列表
  - **Then** 所有任务状态为"已完成"，顶部显示"今日任务全部完成！🎉"

- [ ] **AC-104**: 第二天所有任务重置
  - **Given** 用户昨天完成了所有任务
  - **When** 今天查看任务列表
  - **Then** 所有任务进度归零，状态为"进行中"

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-201**: 每人每天限签一次 ← BR-001
  - **Given** Redis Key `checkin:{userId}:{today}` 存在
  - **When** 再次请求签到
  - **Then** 返回 409，经验不重复发放

- [ ] **AC-202**: 连续 7 天奖励叠加 ← BR-002
  - **Given** 用户连续签到 7 天
  - **When** 第 7 天签到
  - **Then** 总经验 = 5（基础）+ 10（7 天奖励）= 15

- [ ] **AC-203**: 任务进度与操作同步 ← BR-005
  - **Given** 用户生成一个插件后
  - **When** 检查数据库中今天的 `DailyTask.generate_plugin.progress`
  - **Then** 进度为 1

- [ ] **AC-204**: 所有任务每日重置 ← BR-004
  - **Given** 数据库中的 `DailyTask` 记录日期为昨天
  - **When** 今天查询任务
  - **Then** 返回的任务记录日期为今天，进度为 0
