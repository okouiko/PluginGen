# 性能优化 — 技术设计文档

## 1. 设计概要

**功能描述**：对 PluginGen 进行系统性性能优化——前端代码分割、TanStack Query 缓存调优、虚拟滚动、Monaco Editor 懒加载、Prisma 查询优化、Nginx Gzip/缓存配置、构建产物优化。

**影响范围**：前端（代码结构/构建配置/组件）+ 后端（Prisma 查询）+ 基础设施（Nginx 配置）。

**技术难点**：Monaco Editor 的按需加载需要配合 Vite 的 manualChunks 配置，确保编辑页 chunk 和其他页面 chunk 完全隔离。

**外部依赖**：`@tanstack/react-virtual`（需新增安装）

### 2.4 虚拟滚动

```tsx
// components/community/VirtualPluginGrid.tsx → AC-004
import { useVirtualizer } from '@tanstack/react-virtual';

export function VirtualPluginGrid({ plugins }: { plugins: PluginSummary[] }) {
  const parentRef = useRef<HTMLDivElement>(null);

  const virtualizer = useVirtualizer({
    count: plugins.length,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 200, // 每行高度 200px（含间距）
    overscan: 5, // 额外渲染 5 行缓冲
  });

  return (
    <div ref={parentRef} className="h-[600px] overflow-y-auto">
      <div style={{ height: `${virtualizer.getTotalSize()}px` }}>
        {virtualizer.getVirtualItems().map((virtualItem) => (
          <div
            key={virtualItem.key}
            style={{
              position: 'absolute',
              top: 0,
              left: 0,
              width: '100%',
              height: `${virtualItem.size}px`,
              transform: `translateY(${virtualItem.start}px)`,
            }}
          >
            <PluginCard plugin={plugins[virtualItem.index]} />
          </div>
        ))}
      </div>
    </div>
  );
}
```

### 2.7 Nginx 配置 → AC-007

```nginx
# nginx/sites/plugingen.conf
server {
    listen 80;
    server_name plugingen.example.com;

    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml text/javascript;
    gzip_min_length 1024;
    gzip_comp_level 6;
    gzip_vary on;

    root /app/frontend/dist;
    index index.html;

    # 静态资源 - 带 hash 的文件长期缓存
    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
        access_log off;
    }

    # SPA 路由 - 不缓存 index.html
    location / {
        try_files $uri $uri/ /index.html;
        add_header Cache-Control "no-cache";
    }

    # API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_buffering off;  # 流式响应
    }

    # WebSocket 代理
    location /ws/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

## 4. 技术决策

### 4.1 @tanstack/react-virtual 而非 react-window

**背景**：虚拟滚动库的选择。

**结论**：选择 `@tanstack/react-virtual`。对比 `react-window`，它支持动态高度（`estimateSize`）、粘性头部、滚动恢复，且与 TanStack Query 同属 TanStack 生态。

### 4.2 manualChunks 分离 vendor 而非自动 code splitting

**背景**：Vite 默认已经按动态 import 分割，但 React/Query/Monaco 这些大包需要进一步隔离。

**结论**：使用 `manualChunks` 将 `react`、`@tanstack/react-query`、`monaco-editor` 分离为独立 chunk。这样浏览器可以长期缓存不变的基础库 chunk，只更新业务代码 chunk。

### 4.3 select 限定字段而非全局 Prisma 中间件

**背景**：确保所有查询不用 `SELECT *`。

**选项**：

- A: Prisma 中间件自动排除大字段（如 `compileLog`）— 自动但不可见
- B: 每个查询手动写 `select` — 明确但需要代码审查保障

**结论**：选择 B。手动 `select` 让每次查询的字段选择显式可见，代码审查时一目了然。同时作为开发规范的审查项。
